# Web Services Self Register When Missing

Migration ID: `2026.05.29.2__web_services_self_register_when_missing`

## Purpose

Clarify that config-service self-registration is for web-facing services only,
and that a web service should create its own missing service record at startup.

## Apply

- For projects with a web app, HTTP API, task-manager service, local daemon
  with a port, or another discoverable runtime URL, check config-service during
  process startup when self-registration is enabled.
- On startup, query the app's own `service_id`.
- If the service record is missing, create it with the current port and
  documented endpoints.
- If the service record exists but the port or endpoints changed, refresh it.
- For desktop apps, CLI tools, libraries, scripts, and other non-web
  applications, do not query or publish to config-service during normal startup
  unless local instructions explicitly define a discoverable web/API runtime.

## Verification

- Run `git diff --check`.
- For a web service with self-registration enabled, confirm startup creates a
  missing config-service record and refreshes a stale port.
- For a non-web app, confirm startup does not contact config-service unless a
  local web/API runtime is explicitly documented.

## Rollback Or Conflict Notes

If a project already has stricter service-discovery startup rules, preserve the
local rules while keeping the web-service-only boundary.
