# Harden Routed GI Command Loading

Migration ID: `2026.06.24.1__harden_routed_gi_command_loading`

## Purpose

Prevent agents from missing command-specific GI rules after the runtime
entrypoint was modularized into routed modules.

## Shared Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `templates/AGENTS.template.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`
- `VERSION.md`
- `CHANGELOG.md`

## Project Files

- `AGENTS.md`
- `COMMANDS.md`
- `tools/project-memory/instruction-kit.json`

## Apply

- Treat `gi ...`, `ги ...`, and known mojibake command forms as concrete tasks,
  even when the user's message is otherwise short.
- Read `COMMANDS.md` when present, then read every runtime module routed to the
  requested command before acting.
- Copy the shared `COMMANDS.md` into GI-enabled projects so command aliases and
  execution guards are available locally.
- Do not run state-changing GI commands from memory, old chat examples, or the
  command name alone.
- If the routed runtime module is unavailable, stop and report the missing path.
- For `gi restart` / `gi reboot` and aliases, read
  `patterns/AGENTS_RUNTIME/09-project-operation-commands.md` before any process
  inspection, stop, start, or success report.

## Verify

- Confirm the project-local `AGENTS.md` loading contract contains the GI command
  loading guard.
- Confirm copied projects include `patterns/AGENTS_RUNTIME/09-project-operation-commands.md`.
- For a restart/reboot request, confirm the agent's final report accounts for
  each documented app or runtime with verification evidence.
