# Do Not Require Git For GI Updates

Migration ID: `2026.05.16.12__do_not_require_git_for_gi_updates`

## Purpose

Clarify that `gi` means `general-instructions`, not `git`, and that missing git
metadata should not prevent instruction-kit updates.

## Shared Source Files

- `COMMANDS.md`
- `patterns/INSTRUCTION_KIT_MIGRATIONS.md`
- `patterns/GIT_WORKFLOW.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Apply

- Do not interpret `gi` as `git`; treat it as `general-instructions`.
- A missing `.git` directory must not make the agent conclude that there is
  nothing to update.
- Check and apply instruction-kit file updates in the current project when
  possible, even if the current project is not a git repository.
- Skip only the automatic commit/push step when git or a remote is unavailable.
- Report the skip clearly: the GI update was applied or checked, but commit/push
  was skipped because this project is not a git repository or has no remote.

## Verify

- In a project folder without `.git`, ask `gi обновись`.
- Confirm the agent checks or applies instruction-kit updates instead of saying
  that `.git` is missing and there is nothing to update.
- Confirm the response stays scoped to the GI command and reports commit/push as
  skipped when git is unavailable.
