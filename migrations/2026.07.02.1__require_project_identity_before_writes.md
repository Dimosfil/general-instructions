# Require Project Identity Before Writes

Migration ID: `2026.07.02.1__require_project_identity_before_writes`

## Summary

Add a pre-write guard that makes agents compare the active project root and
local project identity with the user's current target before reading or writing
outside the current repository.

## Scope

- `AGENTS.md`
- `COMMANDS.md`
- `CHANGELOG.md`
- `VERSION.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `patterns/AGENTS_RUNTIME/07-startup-and-scope.md`
- `patterns/AGENTS_RUNTIME/10-private-scope-and-missing-context.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `tools/project-memory/instruction-kit.json`
- `templates/instruction-kit.template.json`

## Accepted Guidance

- Before filesystem writes, verify that the active working directory, local
  project identity, and target path match the user's current request.
- Use local identity signals such as `AGENTS.md`, README title, package or app
  manifests, service id, git remote, project-memory orientation, and documented
  working areas.
- If those signals point to a different project than the request, stop and
  report the mismatch before editing.
- A path or product name from old chat, a screenshot, task-manager metadata, a
  summary, or a stale plan is not permission to edit that other project.
- If the current message explicitly names an absolute path outside the active
  project root and an action, state the active root and external target before
  acting. If the path looks like another project root, proceed only when the
  user clearly asked to work in that external project for this task.
