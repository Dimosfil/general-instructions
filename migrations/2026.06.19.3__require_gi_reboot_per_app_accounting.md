# Require gi reboot per-app accounting

Migration ID: `2026.06.19.3__require_gi_reboot_per_app_accounting`

## Summary

Tighten `gi reboot` / `gi restart` so agents cannot report success after
starting only the web/API runtime when the project also has a desktop app,
worker, or other documented runtime.

## Instructions

- Before starting anything for `gi reboot` / `gi restart`, identify the full
  project app set from local run instructions, manifests, service records,
  desktop packaging metadata, or project memory.
- Do not assume a successful web/API start covers the project.
- If local instructions define a preferred command that starts or restarts the
  full app set, use it.
- Otherwise enumerate every documented app or runtime, such as desktop app,
  web/API app, and background workers, then restart each running app and start
  each missing app in the background.
- Verify each documented startup signal: expected processes, visible desktop
  windows when applicable, health/discovery endpoints for web/API apps, worker
  readiness signals, and relevant startup or crash logs when documented.
- The final report must account for each app by name or role with
  started/restarted/skipped status and verification evidence.
- Do not report success from a PID alone, from a web health check alone, or
  while any expected desktop app, web/API app, or worker is unlaunched or
  unverified.
- If a documented desktop app lacks a launch command or window verification
  signal, report that as a blocker or partial failure instead of success.

## Verify

- Confirm `gi reboot` / `gi restart` guidance requires full app-set discovery
  before launch.
- Confirm success reporting requires per-app accounting.
- Confirm web-only verification is explicitly insufficient when another
  documented app is expected.

## Rollback Or Conflict Notes

If a project intentionally has only one app, the per-app report can contain that
single app. If a project intentionally excludes a desktop app from `gi reboot`,
document that narrower scope in project-local run instructions.
