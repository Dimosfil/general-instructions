# Keep Handoff Summaries Substance Focused

Migration ID: `2026.06.18.1__keep_handoff_summaries_substance_focused`

## Summary

Keep `gi summary` / `gi саммари` output focused on the meaning of the thread
instead of routine command bookkeeping.

## Apply

- Treat handoff summaries as compact thread state, not terminal or git ledgers.
- Include user intent, important decisions, code or architecture changes,
  business/product logic, verification evidence, blockers, and next useful
  context.
- Omit routine command bookkeeping such as successful `gi push`, `gi commit`,
  staging counts, git directives, branch names, push targets, and commit hashes
  when those facts are recoverable from git logs or command history.
- Mention repository state only when it changes the next agent's action, such
  as uncommitted work, unresolved conflicts, failed pushes, or a required
  follow-up.
- Keep any needed step-by-step protocol separate as `Thread Timeline` instead
  of mixing it into the ordinary handoff summary.
- Update local summary templates to use substance-oriented sections rather than
  `Commands Run` and unconditional `Git Status Snapshot` sections.
