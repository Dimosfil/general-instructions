Migration ID: `2026.06.21.8__extract_architecture_code_quality_pattern`

only as long inline guidance.

- Project-local `AGENTS.md`
- `patterns/ARCHITECTURE_AND_CODE_QUALITY.md`
- Instruction-kit metadata that records copied files and applied migrations
- Architecture, coding-standard, or working-agreement docs when present

## Rule To Add Or Reference

Add `patterns/ARCHITECTURE_AND_CODE_QUALITY.md` to the copied instruction kit.
In `AGENTS.md`, keep a concise baseline rule and reference the pattern:

```markdown
Build applications with clear architecture and code-quality boundaries. Apply
OOP, SOLID, DRY, clean-code, maintainability, and extensibility principles where
they fit the stack. Keep domain/product logic, orchestration, UI, persistence,
filesystem, external services, and configuration in separate layers with
explicit contracts. Follow `patterns/ARCHITECTURE_AND_CODE_QUALITY.md`.
```

## Pattern Contents

The pattern should cover:

- OOP, SOLID, DRY, clean-code, maintainability, and extensibility expectations;
- separation of domain/product logic, orchestration, UI, persistence,
  filesystem, external services, and configuration;
- interfaces, adapters, ports, repositories, gateways, protocols, typed or
  validated contracts, and dependency inversion where they fit the stack;
- non-OOP equivalents through cohesive modules, functions, services, protocols,
  and data contracts;
- abstraction discipline: add abstractions only when they remove meaningful
  duplication, protect a boundary, or match an established local pattern;
- review and verification checklists for architecture-sensitive work.
