# Retire dry-run for gi test

## Summary

Make `gi test` a live full-system verification command only. Dry-runs,
simulations, dispatcher-only runs, replayed logs, mock-only checks, and
compile/unit-only checks must not be run for `gi test` unless the user
explicitly asks for that diagnostic mode, and they must never satisfy a fresh
`gi test`.

## Scope

- `COMMANDS.md`
- `patterns/PROJECT_TESTING_STRATEGY.md`
- `patterns/AGENTS_RUNTIME/09-project-operation-commands.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`
- `VERSION.md`
- `CHANGELOG.md`

## Rules

- Treat `gi test`, `ги тест`, `gi full test`, `gi release test`,
  `gi system test`, and equivalent wording as a request for a live full-system
  verification run against the active test task.
- Do not use `--dry-run`, simulation mode, dispatcher-only execution, replayed
  logs, mock-only runs, or compile/unit-only checks as the result of `gi test`.
- Do not run dry-run mode during `gi test` unless the user explicitly asks for
  that diagnostic mode.
- If explicitly requested, diagnostic dry-runs must be labeled as diagnostics,
  not as passed verification.
- A valid `gi test` must exercise the documented live runtime surface for the
  selected task: application processes, backend/API, storage, queues or
  workers, UI/auth flows, service discovery, orchestrator or agent handoff
  loops, and health/contract endpoints when the project defines them.
- If the live app/service/worker/UI system cannot be started or reached, report
  `gi test` as blocked or not checked and name the missing live contract.
- Local static, compile, unit, isolated integration, and dispatcher checks are
  useful fast checks, but they do not replace the live system run.

## Verification

- Confirm `patterns/PROJECT_TESTING_STRATEGY.md` rejects dry-run as a valid
  `gi test` result.
- Confirm `patterns/AGENTS_RUNTIME/09-project-operation-commands.md` requires
  live runtime surface coverage.
- Confirm `COMMANDS.md` documents the run-vs-diagnostic distinction.
- Run `git diff --check`.

## Rollback

Remove only the dry-run prohibition and full live-system wording added by this
migration, then restore the previous version metadata.
