# Add Local Sprint Command

Migration ID: `2026.06.24.6__add_local_sprint_command`

## Purpose

Add an explicit local sprint workflow for projects that want sprint-shaped work
without a configured task manager or config-service, while preserving
`gi start sprint` as a manager-backed Sprint/Cycle workflow.

## Apply

- Accept `gi local sprint`, `gi sprint local`, `gi локальный спринт`,
  `gi спринт локально`, and equivalent explicitly local sprint wording as
  requests to run a local sprint checklist.
- Use sprint content from the current message, current chat context, or a
  project-local checklist location documented by local instructions.
- If no sprint content is available, ask one short question for the sprint goal
  and task list.
- Do not resolve config-service, create raw task-manager intake, edit
  task-manager internals, or claim that a visible manager-backed Sprint/Cycle
  was created, started, completed, or synchronized.
- Keep `gi start sprint`, `gi sprint start`, and equivalent active-sprint
  wording routed through the configured task manager. If that setup is missing,
  stop with the exact blocker and mention `gi local sprint` as the explicit
  local alternative.
