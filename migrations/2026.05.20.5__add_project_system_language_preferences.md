## Purpose

language preferences.

## Merge Rules

- Treat `gi commit language`, `gi коммит язык`, `ги коммит язык`, and the older
  `gi язык коммита` forms as the commit-message language command family.
- Treat `gi system language`, `gi систем язык`, and `ги систем язык` as the
  project agent working-language command family.
- Store commit-message preferences in `tools/project-memory/git-preferences.json`.
- Store agent working-language preferences in
  `tools/project-memory/system-preferences.json`.
- The system language applies to progress updates, final answers, clarifying
  questions, and user-facing explanations.
- The system language does not rewrite code, commands, logs, quoted text, or a
  language explicitly requested for a specific response.
- During startup, read and report the current system-language preference when
  available.

## Verification

- Reread edited instruction files.
- Validate `templates/instruction-kit.template.json`,
  `templates/git-preferences.template.json`, and
  `templates/system-preferences.template.json`.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration adds a separate preference file. It must not change the semantics
of existing commit-message language settings.
