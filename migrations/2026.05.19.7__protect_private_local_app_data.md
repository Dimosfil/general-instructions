# Protect Private Local App Data

## Purpose

Prevent agents from turning product ideas, plans, or task-manager notes into
implicit permission to inspect private local logs, telemetry, and application
databases outside the current project root.

## Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `patterns/RAG_STARTUP_FLOW.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat user-home application data and personal telemetry as private external
  sources.
- Do not read `.codex`, `.cursor`, IDE logs, browser profiles, shell history,
  application SQLite databases, or local app logs outside the project root
  unless the user gives an explicit path and action.
- For log, telemetry, or usage analyzer tasks, prefer mock or sample data by
  default. Ask for permission before inspecting a specific real local file.
- Treat product plans, `apps.txt`, summaries, and task-manager notes as intent
  signals only. They are not permission to read private local data sources.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This rule limits default data access, not explicit user-directed diagnostics.
If the user names a concrete private file and action, the agent may inspect only
that requested target and should keep output compact.
