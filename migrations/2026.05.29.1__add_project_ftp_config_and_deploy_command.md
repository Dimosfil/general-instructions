## Apply

- Treat `gi ftp` and `ги фтп` as requests to upload the current project's
  configured build output to FTP, FTPS, or SFTP.
- Treat `gi ftp config`, `gi ftp конфиг`, and `ги фтп конфиг` as requests to
  create, inspect, or update the project-local FTP/SFTP config without
  uploading.
- Store deploy settings in `tools/deploy/ftp.local.json` or a stricter
  project-local equivalent.
- Keep real hostnames, usernames, passwords, tokens, private keys, and private
  remote paths out of git unless project policy explicitly marks them
  non-secret.
- Prefer `tools/deploy/ftp.local.example.json` only as a redacted example.
- Add `tools/deploy/ftp.local.json` to local ignore rules when it may contain
  private deployment details.

## Verification

- Run `git diff --check`.
- Confirm `gi ftp config` configures FTP/SFTP without uploading.
- Confirm `gi ftp` reads project-local config before asking for details.
- Confirm secrets are not printed in final responses or command logs.

## Rollback Or Conflict Notes

If a project already has a deploy command or config path, preserve the local
contract and map `gi ftp` / `ги фтп` to that existing workflow instead of
adding a second deploy system.
