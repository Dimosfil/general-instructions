# Clarify Structured Memory Versus Semantic Retrieval

## Purpose

Propagate the accepted rule that structured memory and vector retrieval serve
different jobs:

- SQLite or another structured store holds deterministic facts and graphs such
  as paths, symbols, GUIDs, generated identifiers, asset links, reverse
  dependencies, commands, failures, and evidence-backed notes.
- Vector retrieval is a second semantic layer for conceptual search over
  curated notes, summaries, architecture docs, and selected chunks.
- Agents must verify current source files before editing because memory indexes
  can be stale.
