# Add GI Summary Command

Migration ID: `2026.05.16.7__add_gi_summary_command`

## Purpose

Add a short command for writing handoff summaries to project files instead of
only summarizing in chat.

## Shared Source Files

- `COMMANDS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/SUMMARY.template.md`

## Project Files

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/summary/`
- `tools/project-memory/instruction-kit.json`

## Apply

- Treat `gi саммари` and `gi summary` as shared instruction-kit commands.
- On `gi саммари` or `gi summary`, create `tools/summary/` if it is missing.
- Write a new handoff summary file named
  `tools/summary/YYYY-MM-DD_HH-mm-ss_AGENT_WORK_SUMMARY.md`.
- Use the structure from `templates/SUMMARY.template.md`: current state, what
  changed, commands run, verification, known caveats, next steps, and git status.
- Do not satisfy the command only by replying in chat.

## Verify

- Confirm `gi саммари` creates a file under `tools/summary/`.
- Confirm the summary is concise and does not include full diffs, large logs,
  secrets, credentials, or private data.
