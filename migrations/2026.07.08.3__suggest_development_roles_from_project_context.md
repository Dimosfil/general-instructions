# Suggest Development Roles From Project Context

Migration ID: `2026.07.08.3__suggest_development_roles_from_project_context`

## Summary

Require agents to infer and briefly propose the most useful lead role or
smallest role set when a development effort begins, based on current project
context and the requested work.

## Instructions

- After initial context loading for a development effort, infer the most useful
  lead role from the project goal, technology stack, README or runbook, project
  memory, manifests, and the user's requested work.
- Briefly propose the role or smallest useful role set before the first
  implementation plan when it improves direction or review quality.
- If the best role is obvious and low-risk, state it as an operating assumption
  and continue.
- Ask only when materially different role choices would change scope,
  architecture, external systems, cost, data safety, or user-visible behavior.
- Revisit the selected role after meaningful pivots, such as moving from
  backend to frontend, design to deployment, or implementation to security
  review. Do not repeat role suggestions on every minor task in the same
  thread.
