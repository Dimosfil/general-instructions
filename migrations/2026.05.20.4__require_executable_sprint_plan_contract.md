# Require Executable Sprint Plan Contract

## Purpose

Prevent agents from sending a sprint plan as raw intake and then assuming the
manager created an executable sprint.

## Source Files

- `skills/task-manager-plans/SKILL.md`
- `skills/task-manager-plans/references/managers/worknest.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `skills/task-manager-plans/SKILL.md`
- `skills/task-manager-plans/references/managers/worknest.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Do not send `kind: sprint-plan` as `type: raw` and report the receipt as an
  executable sprint.
- Before sending sprint work to WorkNest, verify `/agent-intake/contract`.
- Use only the documented executable plan payload for sprint creation, such as
  `type: plan`, `project`, `title`, and `items[]`, unless the project-local
  contract documents a newer equivalent.
- For WorkNest, normalize compact sprint requests into `type: "plan"` with
  non-empty `items[]`; map `planned_changes` to item titles/bodies and
  `verification` to item completion criteria before sending.
- If the contract only stores the payload as raw intake or does not document
  executable sprint creation, stop and report the manager contract mismatch.
- Do not add compatibility code in the target project as a substitute for
  following the manager protocol unless the user explicitly asks to change the
  manager implementation.

## Verification

- Reread edited task-manager skill and WorkNest adapter guidance.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration tightens sprint-plan intake only. It does not remove raw intake
for notes, task candidates, ideas, or other non-executable manager receipts.
