# Migration: Use GitHub Source Repo For Instruction Updates

Version: `2026.06.08.2`

Apply this migration to copied instruction kits that should use the canonical
GitHub repository as the shared-instruction source instead of depending on a
machine-local folder.

## Changes

- Treat `https://github.com/Dimosfil/general-instructions.git` as the canonical
  shared-instruction source repo.
- Store `update_check.source_repo` in `tools/project-memory/instruction-kit.json`.
- Treat `update_check.shared_library_path` as optional local checkout/cache
  metadata, not as the canonical source.
- Update copied `tools/check-instruction-kit-updates.ps1` so it can clone or
  fetch the canonical repo into a local cache when no usable checkout is already
  available.
- Update copied `tools/agent-start.ps1` so startup notices can read an existing
  repo cache without requiring network work during ordinary startup.
- Keep `updates/` maintenance-only; consuming projects still read only accepted
  release artifacts from the repo checkout/cache.

## Verification

- Run `git diff --check`.
- Validate `templates/instruction-kit.template.json` and any edited
  `tools/project-memory/instruction-kit.json` with `ConvertFrom-Json` or another
  JSON parser.
- Run the copied update-check script or template against the canonical repo when
  network and git are available.
- Confirm no secrets, generated logs, private project data, or unrelated changes
  are included.
