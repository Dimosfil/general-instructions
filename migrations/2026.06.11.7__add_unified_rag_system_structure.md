# Add Unified RAG System Structure

Migration ID: `2026.06.11.7__add_unified_rag_system_structure`

## Summary

Add a reusable RAG structure that lets projects grow from Markdown and SQLite
FTS into semantic retrieval through Chroma, Qdrant, pgvector, or another backend
without changing agent-facing workflows.

## Accepted Guidance

- Treat RAG as a layered system, not as a synonym for vector search.
- Keep source corpus rules, exclusions, manifest metadata, chunk metadata,
  structured memory, retrieval adapters, context packets, observability, evals,
  and writeback as separate responsibilities.
- Keep Chroma, Qdrant, pgvector, and similar stores behind a retrieval adapter
  contract so prompts, `gi` startup, and memory writeback do not depend on one
  vector database.
- Use SQLite FTS for exact paths, commands, symbols, and error text before
  adding semantic retrieval.
- Add Chroma for a local semantic MVP; move to Qdrant or pgvector only when
  service-grade operation, shared access, stronger filtering, snapshots, or
  PostgreSQL integration justify it.
- Keep generated vector indexes ignored when rebuildable, and commit only
  reviewable config, docs, schemas, and scripts.
- Keep `tools/project-memory/rag-system.json` free of secrets, private runtime
  endpoints, credentials, and local-only sensitive paths.
