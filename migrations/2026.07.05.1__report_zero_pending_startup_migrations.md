# 2026.07.05.1 - Report Zero Pending Startup Migrations

## Summary

Clarify the first-concrete-message GI update check reporting contract. The
compact startup status must explicitly include the pending migration count,
including `0` when no migrations are pending.

## Applicability

Apply to copied instruction kits that use the quiet startup GI update check.

## Changes

- Update `AGENTS.md` and copied templates so the quiet GI update check reports
  the pending migration count.
- Update `patterns/AGENTS_RUNTIME/07-startup-and-scope.md` so task-routed
  startup behavior requires the count in the compact status line.
- Update `COMMANDS.md` so `gi` command documentation matches the runtime rule.
