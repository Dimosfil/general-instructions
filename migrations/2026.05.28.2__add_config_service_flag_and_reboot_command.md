# Add Config Service Flag And Reboot Command

Migration ID: `2026.05.28.2__add_config_service_flag_and_reboot_command`

## Purpose

Separate config-service URL configuration, application self-registration, and
application runtime control into distinct commands.

## Shared Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `patterns/SERVICE_DISCOVERY_CONFIG.md`
- `templates/AGENTS.template.md`
- `templates/config-service-local.template.json`
- `migrations/2026.05.28.2__add_config_service_flag_and_reboot_command.md`

## Project Files

- `AGENTS.md`
- project-local config-service/app runtime config, if documented
- local run instructions for starting or restarting the application

## Apply

- Treat `gi config service url=<url>` as the command for setting the
  config-service URL.
- Treat `gi config service on` and `ги конфиг сервис on` as commands to enable
  the current application's self-registration in config-service.
- Treat `gi config service off` and `ги конфиг сервис off` as commands to
  disable the current application's self-registration in config-service.
- Store the self-registration flag alongside the project's documented
  config-service URL or run instructions, not in GI main config.
- When enabling self-registration, confirm a config-service URL is already
  configured in that same local config area or documented GI bootstrap config.
  If no URL is configured, stop and tell the user to set
  `gi config service url=<url>` first.
- Do not reinterpret `on` or `off` as starting or stopping the config-service
  process.
- Treat `gi reboot`, `ги ребут`, `gi restart`, and `ги рестарт` as requests to
  start or restart the current application using project-local run
  instructions. If the app is running, restart it; otherwise start it in the
  background.

## Verify

- Confirm `on/off` changes only the current app's config-service
  self-registration flag.
- Confirm `url=<url>` remains the URL configuration command.
- Confirm `gi reboot` uses local app run instructions and does not manage the
  config-service process unless the current app is the config-service itself.

## Rollback Or Conflict Notes

If a project has no documented location for the self-registration flag or no
run command for reboot, ask one short clarification question instead of
inventing hidden config files or process-management rules.
