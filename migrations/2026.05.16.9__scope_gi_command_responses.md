# Scope GI Command Responses

Migration ID: `2026.05.16.9__scope_gi_command_responses`

## Purpose

Keep short `gi` command responses focused on shared instruction-kit work and
prevent agents from automatically resuming older product tasks afterward.

## Shared Source Files

- `COMMANDS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Apply

- Treat `gi ...` as scoped shared instruction-kit commands.
- After completing a `gi` command, summarize only that command's result and
  stop.
- Do not automatically return to an older product task, previous question, or
  unrelated implementation plan after a `gi` command.
- Continue prior product work only when the user explicitly asks to continue.

## Verify

- Ask `gi обновить` or `gi обновись`.
- Confirm the response reports only the instruction-kit update result.
- Confirm it does not append analysis or next steps for an older product task.
