# Add GI Git Finish Commands

Migration ID: `2026.05.18.4__add_gi_git_finish_commands`

## Purpose

Add short `gi` commands for commit-only, push-only, and commit-then-push
workflows.

## Apply

- Treat `gi коммит` and `gi commit` as explicit requests to commit scoped
  current changes without pushing.
- Treat `gi пуш`, `gi коммит пуш`, `gi commit push`, `gi commit and push`, and
  `gi finish` as explicit requests to commit scoped current changes, then push
  the current branch.
- Treat `gi только пуш`, `gi push only`, and `gi just push` as explicit requests
  to push existing local commits without staging files or creating a new commit.
- Before committing or pushing, inspect `git status --short`, staged and
  unstaged changes, current branch, and remote/upstream configuration.
- Stage only files that belong to the current task or explicitly requested
  scope.
- Follow `tools/project-memory/git-preferences.json` for commit-message language
  preferences when creating a commit.
- Stop and explain the blocker if unrelated/user changes would be included,
  scope is ambiguous, conflicts are present, secrets may be included, there is
  no git repository, no remote/upstream is configured for a push, or push fails.

## Verify

- Ask `gi коммит` and confirm the agent commits scoped changes without pushing.
- Ask `gi пуш` or `gi коммит пуш` and confirm the agent commits scoped changes
  and pushes the current branch.
- Ask `gi только пуш` and confirm the agent pushes existing local commits
  without creating a new commit.
