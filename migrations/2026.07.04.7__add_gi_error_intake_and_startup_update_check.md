# Migration: Add GI Error Intake And Startup Update Check

Migration ID: `2026.07.04.7__add_gi_error_intake_and_startup_update_check`

## Purpose

Make recurring GI rule bugs easier to report and repair, and make new chat
sessions start from the latest accepted instruction-kit migrations without
large context reads.

## Apply

- Add `gi ошибка`, `ги ошибка`, and `gi error` as GI rule-error intake commands.
  The agent collects only evidence already available in the current chat,
  attached screenshots/files, visible tool output, and explicitly authorized
  local paths.
- For intake, append or prepare a compact entry in
  `updates/USER_REPORTED_AGENT_BUG_LOG.md` in the shared instruction library
  when available, or in a project-local intake folder when it is not.
- Intake entries should include date, symptom, likely violated rule or rule gap,
  evidence summary, privacy review, status, and any migration or follow-up.
- Intake does not authorize fixing shared rules, inspecting unrelated projects,
  reading private paths, or running broad searches.
- Add `gi ошибка фикс`, `ги ошибка фикс`, and `gi error fix` as the repair
  command. It authorizes the agent to read the newest relevant unresolved
  bug-log entry plus current evidence, extract the portable rule gap, update
  live rules, copied-project templates, accepted migrations, version, changelog,
  and bug-log status, and verify the scoped change.
- On the first concrete task in a new chat/session, run a quiet GI update check
  before task-specific startup restore, planning, implementation, or command
  execution.
- The startup update check reads local instruction-kit metadata and accepted
  source `VERSION.md`, `CHANGELOG.md`, `INDEX.md`, and pending `migrations/`
  files only. It must not read `updates/`, old chat examples, broad project
  files, or unrelated source repositories.
- Apply pending accepted migrations when the local update contract allows it.
  If the source is unavailable or a migration is blocked, report a compact
  blocker and continue with current local instructions unless the user
  explicitly requested `gi обновить`.

## Verify

- Confirm `COMMANDS.md` lists `gi ошибка` and `gi ошибка фикс` with separate
  intake and repair behavior.
- Confirm `patterns/AGENTS_RUNTIME/14-update-intake.md` defines bug intake and
  fix handling.
- Confirm `patterns/AGENTS_RUNTIME/07-startup-and-scope.md`, root `AGENTS.md`,
  and `templates/AGENTS.template.md` define the first-concrete-message quiet GI
  update check.
- Confirm `VERSION.md` and `CHANGELOG.md` mention this migration.

## Rollback Or Conflict Notes

This migration does not require consuming projects to read `updates/` during
normal startup. `updates/` remains maintenance-only intake; startup checks use
accepted version and migration artifacts only.
