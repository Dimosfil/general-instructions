# Reconcile Handoff With Thread Conclusion

Migration ID: `2026.06.18.2__reconcile_handoff_with_thread_conclusion`

## Summary

When answering where a previous thread stopped, reconcile the handoff summary
with the latest visible thread conclusion or user-provided evidence. Do not
treat incidental caveats in a summary as the active next task.

## Apply

- Treat handoff summaries as evidence, not the sole authority, when the user
  asks where a previous thread stopped.
- Compare the summary with the latest visible thread conclusion, screenshots,
  direct quotes, or other user-provided evidence.
- Prefer the last explicit architectural/product decision, open question, or
  agreed next direction over incidental caveats or old next-step bullets.
- Do not promote an unverified caveat, environment variable, skipped check, or
  old `Next Best Steps` item into the current task unless the user selects it or
  it blocks the stated goal.

## Verify

- Ask "where did we stop?" with a summary that mentions an unverified caveat and
  a newer thread conclusion that points elsewhere. Confirm the answer follows
  the newer conclusion and mentions the caveat only as secondary context.
