# Add GI Build Command

## Summary

Add `gi build` / `gi собрать` / `ги билд` / `ги собрать` as the clear command
for producing the current project's pre-release distributable or upload-ready
build output.

## Instructions

- Treat `gi build`, `gi собрать`, `ги билд`, `ги собрать`, `gi rebuild`, and
  `ги ребилд` as requests to build or rebuild only the current project or
  application.
- Produce the documented release/upload-ready artifact, such as a static
  `dist/`, bundle, package, executable, or other project-local build output.
- Read project-local build or rebuild instructions, manifests, scripts, hosting
  base-path/public-path config, and packaging metadata before running the
  documented command.
- Do not treat these commands as dependency restore, tests-only verification,
  FTP/SFTP upload, production publication, installer packaging, or a GI/RAG
  tooling rebuild.
- If no project build/rebuild contract exists, ask one short clarification
  question instead of inventing a command.
- Use `gi ftp` for upload, `gi prod` for documented production publication,
  `gi install` for installer packaging, and `gi tools rebuild` or
  `gi rag rebuild` for GI/RAG rebuilds.

## Verify

- Confirm `COMMANDS.md` lists `gi build`, `gi собрать`, `ги билд`,
  `ги собрать`, `gi rebuild`, and `ги ребилд` as project build/rebuild
  commands.
- Confirm `patterns/AGENTS_RUNTIME/09-project-operation-commands.md` keeps the
  project build command separate from upload, production publication,
  installer packaging, and GI/RAG rebuild commands.
- Confirm `AGENTS.md` and `templates/AGENTS.template.md` route state-changing
  project build/rebuild commands to
  `patterns/AGENTS_RUNTIME/09-project-operation-commands.md`.
- Confirm `templates/AGENT_WORKING_AGREEMENTS.template.md` includes the new
  aliases for copied project instructions.

## Rollback Or Conflict Notes

If a consuming project already has a local `gi build` meaning, project-local
instructions may override this shared rule, but the local command must still
document whether upload, production publication, installer packaging, or GI/RAG
rebuilds are included.
