# Tighten Project Study And Nested Scope

## Purpose

Prevent agents from doing broad first-pass repository scans, over-reading nested
third-party trees, or duplicating automatic UI tool counters in progress
messages.

## Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `patterns/RAG_STARTUP_FLOW.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Add first-pass project study guidance:
  - read local instructions, README, manifests, and config entry points first;
  - use recursive scans only after targeted search fails or the task clearly
    requires repository-wide inventory.
- Treat nested checkouts, vendored repositories, cloned examples, and
  third-party source trees as separate scope. Inspect them only when the user
  explicitly asks, the task is about that nested tree, or local instructions
  identify it as an active workspace component.
- Clarify that automatic chat UI tool counters are not agent progress updates.
  Agents should not duplicate those counters in prose.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration narrows default exploration behavior. It should not block
explicit repository-wide audits, release checks, incident debugging, or tasks
that genuinely require broad inventory.
