# Migration: Centralize Deploy In Devops Gateway

Migration ID: `2026.07.04.8__centralize_deploy_in_devops_gateway`

## Purpose

Retire project-owned personal deploy paths from ordinary GI-enabled projects and
keep direct deploy/FTP/SFTP scripts only in projects explicitly marked as deploy
infrastructure.

## Apply

- Add `gi set devops`, `gi devops`, `ги девопс`, and equivalent wording as the
  command that marks the current project as a deploy-infrastructure owner.
- The command verifies the active project identity, then creates or updates an
  ignored local marker such as `tools/project-memory/devops.local.json` with
  non-secret metadata: role `deploy-owner`, timestamp, reason, and optional
  deploy entrypoint or gateway contract pointer.
- During this migration, ordinary non-devops projects remove, disable, or stop
  relying on direct deploy/upload scripts, FTP/SFTP configs, and private deploy
  helpers. They keep only selected-gateway metadata, redacted examples, and
  documented build artifact contracts needed by the gateway.
- Non-devops `gi ftp`, `gi ftp push`, `gi upload ftp`, and `gi deploy ftp`
  requests must use a saved or user-provided deploy gateway. If no gateway is
  selected, stop and ask for the gateway path instead of creating a new personal
  FTP/SFTP deploy path.
- Projects marked devops keep and maintain direct deploy scripts/configuration
  as gateway-owned infrastructure. As a temporary local rollout note, the
  current deploy-owner project can be marked by running `gi set devops` in the
  existing deploy gateway checkout explicitly named by the user.

## Verify

- Confirm `patterns/AGENTS_RUNTIME/09-project-operation-commands.md` defines
  `gi set devops` and the non-devops deploy cleanup behavior.
- Confirm `patterns/PROJECT_FTP_DEPLOY.md` no longer lets ordinary non-devops
  projects fall back to personal direct `tools/deploy/ftp.local.json` upload.
- Confirm `COMMANDS.md` lists `gi set devops` in the command examples, help
  table, and production/deploy command section.
- Confirm `.gitignore` and `templates/gitignore-agent-memory.template` ignore
  `tools/project-memory/devops.local.json`.
- Confirm `VERSION.md`, `CHANGELOG.md`, and instruction-kit metadata mention
  this migration.

## Rollback Or Conflict Notes

This migration should not delete deploy infrastructure from a project marked
devops. If a consuming project still needs a temporary direct deploy for legacy
compatibility, document that as a deploy-gateway delegation or ask the user for
an explicit temporary exception instead of silently keeping personal deploy
scripts.
