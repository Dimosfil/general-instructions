# Number Unified Project Language Selection

Migration ID: `2026.05.27.4__number_unified_project_language_selection`

## Purpose

Restore the old numbered-language-selection ergonomics for the unified
`gi language` / `gi язык` flow. A user reply such as `1 2` should be accepted
after the agent has shown a numbered language checklist.

## Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- If `gi язык`, `ги язык`, `gi language`, or an equivalent unified
  project-language command is sent without explicit languages, ask in three
  sequential chat steps:
  1. project working environment languages;
  2. commit-message languages;
  3. task languages.
- At each step, show the same concise numbered Markdown checklist of available
  languages, with the current selection checked.
- Tell the user they may answer with numbers or language names in priority
  order.
- If the user replies with only numbers, such as `1 2`, map the numbers to the
  most recent checklist and preserve that order for the current step.
- Do not ask what the numbers mean when the numbered checklist was just shown.
- Keep direct inline forms such as `gi язык: 2 1` working as a single selection
  applied to all three surfaces unless separate values are supplied.

## Verification

- Ask `ги язык` without languages.
- Confirm the agent shows the first numbered checklist and accepts `1 2` as the
  ordered language choice for that step.
- Confirm the agent continues to steps 2 and 3 and writes
  `tools/project-memory/system-preferences.json` and
  `tools/project-memory/git-preferences.json`.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration changes only chat handling instructions for the unified language
flow. It does not change the preference file schema.
