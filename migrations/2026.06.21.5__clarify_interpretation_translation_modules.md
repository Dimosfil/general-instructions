# Clarify Interpretation Translation Modules

Migration ID: `2026.06.21.5__clarify_interpretation_translation_modules`

Apply this migration to copied instruction kits that already adopted query and
prompt normalization boundaries. The key clarification is that multilingual
search, query understanding, prompt expansion, and model-facing query creation
belong in an interpretation/translation capability, not merely in static
dictionary files.

## Project Files To Update

- Project-local `AGENTS.md`
- Search/retrieval/ranking/model-adapter architecture docs
- Query/prompt normalization resources or adapter docs
- Project-memory specs for intent interpretation, translation, search, or
  model-facing query construction when present

## Rule To Add

- Treat query interpretation and translation as a dedicated module or capability.
- The implementation may be deterministic resources, curated dictionaries, local
  algorithms, retrieval-backed components, a local or remote service, or a
  provider-swappable LLM adapter.
- Keep provider-specific LLM calls, prompts, model names, budgets, fallbacks,
  timeouts, and privacy policy behind an interface or adapter. Do not embed them
  in product/UI/request logic.
- Preserve the original user query separately from interpreted intent and
  model-facing queries.
- Tests should prove that the interpretation implementation can change without
  rewriting product/UI logic.

## Verification

- Search changed code for inline intent heuristics, provider-specific LLM calls,
  hard-coded model names, prompt strings, and dictionaries in UI/request code.
- Confirm interpretation/translation behavior is behind an interface, resource,
  adapter, service, or pipeline step.
- Confirm provider changes are configuration or adapter changes, not product/UI
  rewrites.
- Test at least one original query, interpreted intent, and model-facing query.

## Rollback

Remove only this clarification. Do not reintroduce inline dictionaries,
provider-specific LLM calls, or prompt-expansion logic into product/UI/request
code.
