# Require Startup Migration Auto-Application

Migration ID: `2026.07.14.1__require_startup_migration_auto_apply`

## Purpose

Prevent first-task startup checks from reporting a newer accepted GI version
without applying its pending migrations.

## Accepted Guidance

- Treat `update_check.enabled: true` as authorization to resolve the accepted
  source and apply pending accepted migrations before task-specific work.
- Treat a missing `auto_apply_pending_migrations` field as `true` for backward
  compatibility with installed instruction kits.
- Do not stop at an update-availability notice or defer application to a later
  `gi update` command.
- Skip automatic application only for an explicit `false` setting or a named
  concrete blocker such as unavailable source, read-only files, unsafe scope,
  unrelated dirty-file overlap, or a merge conflict.
- Apply and verify instruction files before recording migration metadata.
