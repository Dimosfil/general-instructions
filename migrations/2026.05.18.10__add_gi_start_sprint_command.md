# Add GI Start Sprint Command

Migration ID: `2026.05.18.10__add_gi_start_sprint_command`

## Purpose

## Apply

- Treat `gi старт спринт` as the command to restore context, find the active
  sprint in the configured task manager, and begin executing its tasks.
- Accept aliases: `gi start sprint`, `gi sprint start`,
  `gi активный спринт`, and `gi work sprint`.
- For WorkNest-style Markdown projects, find active sprints through `sprint.md`
  files, then execute numbered task files in ascending `Order`.
- If exactly one active sprint exists, start it without asking. If none or many
  exist, ask the user to choose.
- Update each task file to `in_progress` before execution and `done` with a
  concise completion/result section after successful completion.
- Continue through `todo` and `ready` tasks until the sprint is complete or a
  blocker requires user input.
- Stop before destructive actions, missing access, conflicts, or verification
  failures that need user attention.

## Verify

- Confirm `gi старт спринт` is listed in `COMMANDS.md`.
- Confirm WorkNest adapter guidance describes active sprint discovery and
  ordered task execution.
