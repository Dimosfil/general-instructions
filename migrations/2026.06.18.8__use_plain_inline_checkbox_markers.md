# Use plain inline checkbox markers

Accepted version: `2026.06.18.8`

## Purpose

Keep checkbox-like selection prompts visually aligned in chat clients that split
Markdown task-list controls from their labels.

## Problem

Some chat renderers split Markdown task-list syntax such as
`- [x] 1. English` into a checkbox control on one visual row and the label on a
separate row. Keeping the marker, number, and label on the same source line is
not enough in those renderers.

## Apply

- For numbered selection prompts in chat, use plain inline checkbox markers:

```markdown
[x] 1. English
[x] 2. Russian
[ ] 3. Spanish
[ ] 4. German
[ ] 5. French
```

- Do not use Markdown task-list syntax such as `- [x] 1. English`.
- Do not use ordered-task syntax such as `1. [x] English`.
- Keep accepting numeric replies against the visible numbers in the most recent
  checklist.
- Ordinary project TODO lists may still use Markdown task-list syntax when they
  are documents rather than chat selection prompts.

## Verification

- Show a language, task-manager, FTP service, or folder-choice prompt in chat.
- Confirm each marker, number, and label renders on one visual row.
- Run `git diff --check`.
