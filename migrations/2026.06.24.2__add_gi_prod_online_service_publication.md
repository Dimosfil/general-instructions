## Apply

- Add `gi prod`, `gi production`, `gi прод`, and `ги прод` as production service
  publication commands.
- Use the command only for online services connected to live remote APIs,
  webhooks, chats, marketplaces, payment providers, or similar external systems.
- Keep normal development, refactoring, tests, cleanup, formatting, and
  `gi restart` on the development checkout/service.
- Do not edit, reset, test, stop, or restart the production service folder
  unless the user explicitly invokes the production workflow or stricter local
  instructions require it.
- Require a project-local production contract: production folder, include and
  exclude rules, production-local secrets/config/state to preserve,
  restart/reload/switchover command, health check, and rollback strategy.
- Preserve production-local secrets, credentials, databases, sessions, logs,
  caches, webhook/API state, service manager files, and user data.
- Never copy production secrets or runtime state back into development.
- Add `tools/deploy/prod.local.json` to ignored local agent/deploy state when
  projects use that file for machine-specific production settings.
