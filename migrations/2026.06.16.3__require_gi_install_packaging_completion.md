# Require GI Install Packaging Completion

Migration ID: `2026.06.16.3__require_gi_install_packaging_completion`

## Summary

Clarify that `gi install` is not complete after dependency restore, build, or
tests alone. Agents must run the project-local packaging command and verify a
current installer artifact before reporting success.

## Scope

- `AGENTS.md`
- `COMMANDS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `tools/project-memory/instruction-kit.json`
- `templates/instruction-kit.template.json`
- `VERSION.md`
- `CHANGELOG.md`

## Accepted Guidance

- Treat `restore`, dependency install, build, and test as preliminary checks for
  `gi install`, not as completion of the installer request.
- Complete `gi install` only after the packaging command runs and a current
  installer artifact is produced or explicitly verified.
- Resolve the installer version from project metadata or the local installer
  contract before packaging; do not invent a version.
- If only verification checks ran, say so plainly and do not describe the
  project as installed or restored.
- On success, report the installer artifact path, version, and checks.

## Verification

- Ask `gi install` in a project with an existing packaging script; confirm the
  agent runs that script and reports the installer artifact.
- Ask `gi install` in a project where restore/build/test pass but packaging is
  not run; confirm the agent does not claim installer success.
- Ask `gi install` where version metadata or installer contract is missing;
  confirm the agent asks one concise clarification question instead of
  inventing a version or installer configuration.
- Run `git diff --check`.
