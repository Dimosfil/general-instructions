# Add Task Manager Lifecycle Test Command

## Purpose

Add a reusable command for testing the currently configured task manager through
the full agent lifecycle: task creation, loading, reading, execution, completion,
and final status readback.

## Source Files

- `COMMANDS.md`
- `skills/task-manager-plans/SKILL.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `skills/task-manager-plans/SKILL.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat `gi manager test`, `gi tm test`, `gi манагер тест`, `gi менеджер тест`,
  and equivalent wording as requests to test the current configured task manager
  end to end.
- Stay in the current project root and use only the configured manager API or
  connector.
- If no manager is enabled, run the same manager selection flow as `gi tm`.
- Verify required capabilities before creating work: create/load task,
  next-task or task lookup, status update or start, completion, and final
  readback.
- Create a clearly labeled disposable no-op task that requires no repository
  edits, secret access, destructive action, or cross-project filesystem access.
- Load or read the task back, take it in work when supported, perform a no-op
  verification step, mark it `done` or the adapter's completed equivalent, and
  read the final status back.
- Report manager id, endpoint, task id or URL, lifecycle steps completed, final
  status, and unsupported capabilities.
- If a step fails, stop at the first contract gap, leave the task in the safest
  available state, and report the exact missing or failing lifecycle operation.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration introduces a non-destructive lifecycle test. It must not be used
to close real user work, mutate unrelated manager data, or inspect another
project folder from task-manager metadata.
