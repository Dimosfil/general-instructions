# Migration: Protect RAG Cleanup Assets

Version: `2026.06.11.1`

Apply this migration to copied instruction kits that need safer repository
cleanup and GitHub-preparation behavior.

## Changes

- Add repository cleanup guidance to project `AGENTS.md`:
  - do not remove `AGENTS.md`, `tools/`, `tools/project-memory/`, `skills/`,
    bootstrap scripts, update scripts, deploy scripts, or agent-facing
    instruction/config files merely because they look internal;
  - inspect their purpose first and treat them as possible RAG/startup
    infrastructure;
  - delete them only when the user explicitly confirms they are temporary or
    unrelated to the project.
- Clarify SQLite/database cleanup:
  - classify `*.sqlite`, `*.sqlite3`, and `*.db` before deleting or committing;
  - keep generated agent-memory indexes ignored when they are rebuildable;
  - commit reviewable README files, Markdown/JSON memory exports, schemas, and
    indexing scripts when useful;
  - never commit databases containing secrets, private data, telemetry,
    task-manager state, absolute local paths, or agent conversation history.
- Update local new-project checklists or playbooks if the project keeps copied
  instruction-kit guidance.

## Verification

- Run `git diff --check`.
- If updating JSON instruction-kit metadata, validate it with `ConvertFrom-Json`
  or another JSON parser.
- Confirm no unrelated dirty worktree changes, secrets, logs, local databases,
  or generated caches are included in the commit.
