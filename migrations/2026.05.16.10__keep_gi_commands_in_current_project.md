# Keep GI Commands In Current Project

Migration ID: `2026.05.16.10__keep_gi_commands_in_current_project`

## Purpose

Prevent agents from leaving the current project root when handling short `gi`
commands.

## Shared Source Files

- `COMMANDS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `patterns/INSTRUCTION_KIT_MIGRATIONS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Apply

- Run `gi ...` commands against the current project root.
- Do not switch to another repository, the shared instruction library, or a path
  from an older task unless the user explicitly asks.
- Treat the shared instruction library only as the source of accepted
  instruction-kit artifacts and templates.
- If the current project has no git repository or no instruction-kit metadata,
  report that for the current project and ask what path or init action the user
  wants.

## Verify

- In a project that is not a git repository, ask `gi обновить`.
- Confirm the agent reports the current project's state instead of changing into
  a different repository.
- Confirm it does not use a previous task path as the working project root.
