# Enforce Git Finalization Boundary

Migration ID: `2026.07.13.1__enforce_git_finalization_boundary`

## Summary

Prevent agents from reporting a clean successful commit or push when a
task-scoped tracked file is created or modified after the commit.

## Accepted Guidance

- Complete every task-scoped filesystem write, including handoff, project
  memory, generated metadata, formatting, and verification corrections, before
  staging.
- After the last filesystem mutation and commit/push, run a final
  `git status --short` and verify that the intended commit contains all scoped
  tracked changes.
- For pushes, also verify that the local branch matches its configured
  upstream. Do not treat matching local and remote HEAD values as proof of a
  clean worktree.
- Distinguish pre-existing unrelated dirt from new task-scoped residue. If a
  required file changed after commit, complete the authorized finish workflow
  or report the exact remainder and incomplete status.
- Do not mutate tracked task files after the final status check. If another
  write is necessary, repeat the applicable authorized finish workflow and
  final verification before responding.
