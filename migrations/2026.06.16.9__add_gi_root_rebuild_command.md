# Add GI Root Rebuild Command

## Purpose

Propagate the accepted rule for rebuilding a project's whole configured
project-memory/RAG retrieval system after RAG-impacting changes.

`gi root rebuild` / `gi rag rebuild` / `–≥–∏ —Ä—É—Ç —Ä–µ–±–∏–ª–¥` is a heavy full rebuild
command and requires explicit confirmation immediately before execution. Node
forms rebuild only one configured part, such as SQL, chunks, vector, manifest,
or evals.

RAG-impacting changes and require the affected RAG nodes to be rebuilt under the

## Merge Rules

- Add `gi root rebuild`, `gi rag rebuild`, `–≥–∏ —Ä—É—Ç —Ä–µ–±–∏–ª–¥`, and
  `–≥–∏ —Ä–∞–≥ —Ä–µ–±–∏–ª–¥` as full project-memory/RAG rebuild commands.
- Add node rebuild forms for SQL, chunks, vector, manifest, and evals. Preserve
  stricter project-local names if they already exist.
- Full rebuild must ask for explicit confirmation immediately before running.
  Before asking, list source groups, privacy exclusions, generated paths that
  may be replaced, node commands, status checks, and any required external
  services or dependencies.
- Use only documented project-local commands from `rag-system.json`, runbooks,
  or local helper scripts. If a rebuild node has no command, ask one short
  clarification question instead of guessing.
- Add a `rebuild` object to `tools/project-memory/rag-system.json` when present,
  without overwriting project-specific commands, adapter choices, paths,
  exclusions, or collection names. Include `full_command`, full confirmation
  requirement, node commands/status commands, and rebuild state fields such as
  `last_full_rebuild_migration`, `last_full_rebuild_at`, and
  `last_node_rebuilds`.
- During `gi –æ–±–Ω–æ–≤–∏—Ç—å`, inspect each newly applied migration. If it changes RAG
  source rules, chunking, embedding metadata, SQLite/vector schemas, retrieval
  adapters, or project-memory index scripts, compare the migration id with
  rebuild state. Report stale nodes and ask before a full rebuild, or run/offer
  the smallest documented node rebuild for narrow migrations.
- Do not mark rebuild state current until rebuild and readback/status checks
  succeed. Do not commit generated rebuildable SQLite databases, semantic
  corpora, vector indexes, logs, secrets, telemetry, or private runtime data.

## Verification

- Confirm the project instructions include full and node rebuild command
  handling.
- Confirm full rebuild still requires explicit confirmation.
- Confirm `rag-system.json`, when present, documents rebuild nodes or records
  that a node is intentionally unconfigured.
- Confirm RAG-impacting migrations are checked against rebuild state during
  `gi –æ–±–Ω–æ–≤–∏—Ç—å`.
- Confirm generated indexes and vector stores remain ignored when rebuildable.

## Rollback

Remove only the added rebuild command guidance and `rag-system.json` rebuild
metadata. Leave existing project-memory indexes, scripts, and retrieval
configuration intact.
