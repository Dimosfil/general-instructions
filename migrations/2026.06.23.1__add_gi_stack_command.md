# Add gi stack command

## Summary

Add `gi stack` / `ги стек` as a command for finding or building the current
project's technology stack inventory.

## Scope

- `COMMANDS.md`
- `patterns/TECHNOLOGY_STACK_INVENTORY.md`
- `patterns/AGENTS_RUNTIME/06-tool-usage-and-token-economy.md`
- `templates/TECHNOLOGY_STACK.template.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`
- `VERSION.md`
- `CHANGELOG.md`

## Rules

- Treat `gi stack` and `ги стек` as requests to find or build the current
  project's technology stack inventory.
- First look for a canonical stack link or inventory in project-local
  instructions, README, docs indexes, runbooks, and
  `tools/project-memory/specs/technology-stack.md`.
- For external agents starting outside the project, the first relevant
  project-local instructions or docs should expose a concise pointer to the
  canonical stack inventory near the beginning of the file.
- If a current inventory exists, verify key facts against current manifests,
  lockfiles, config, run instructions, and source entry points before reporting
  it as current.
- If no stack link or inventory exists, gather the stack from current project
  evidence and create or update the canonical inventory, using
  `tools/project-memory/specs/technology-stack.md` unless local instructions
  name a different path.
- Add or request the missing top-level stack pointer according to local
  documentation policy.
- Record unknowns as verification gaps instead of guessing.
- Do not install dependencies, start services, rebuild indexes, call external
  APIs, read secrets, or inspect private paths outside the project root for
  this command unless the user explicitly approves that scope.

## Verification

- Confirm `COMMANDS.md` lists `gi stack` and `ги стек`.
- Confirm `patterns/TECHNOLOGY_STACK_INVENTORY.md` documents the command and
  the visible top-level stack pointer expectation.
- Confirm `templates/TECHNOLOGY_STACK.template.md` has fields for the canonical
  source and inbound link.
- Run `git diff --check`.

## Rollback

Remove only the `gi stack` command guidance, template fields, migration entry,
and version/changelog metadata added by this migration.
