# Require Project-Local Task Manager API Endpoints

## Purpose

Make task-manager setup safer across consuming projects by requiring each
project to store its own manager API endpoint and verify workflow-specific
capabilities before sending plans or starting sprint work.

## Source Files

- `templates/task-managers.template.json`
- `skills/task-manager-plans/SKILL.md`
- `skills/task-manager-plans/references/managers/worknest.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `COMMANDS.md`

## Project Files To Update

- `tools/project-memory/task-managers.json`
- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- copied `skills/task-manager-plans/` files when present
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat task-manager configuration as project-local state, not a global agent
  setting.
- Require each enabled manager to have a real project-specific API endpoint
  before posting plans or fetching sprint/task state.
- Treat `base_url` as the manager API endpoint used for operations.
- Do not use a UI URL as `base_url` unless the adapter explicitly says the UI
  URL is also the API URL.
- Do not leave endpoint fields empty, guessed, or set to `TODO`.
- Before using a manager endpoint, verify the capabilities required for the
  requested workflow, not only generic health.
- If health succeeds but required capabilities fail, report a stale or
  misconfigured manager endpoint and stop before sending work.
- Preserve local manager IDs, project/workspace names, and non-secret
  preferences.
- Do not store secrets, tokens, cookies, passwords, private user data, or
  production data in task-manager config.
- Update `tools/project-memory/instruction-kit.json` after successful file
  changes, not before.

## Verification

- Validate JSON files after editing.
- Reread changed task-manager instructions and config.
- Confirm enabled managers do not keep `base_url: "TODO"`.
- Confirm task-manager guidance requires workflow capability checks.
- Run `git diff --check` when the project is a git repository.

## Rollback Or Conflict Notes

If a local project intentionally uses the same origin for UI and API, keep the
existing endpoint only when the adapter or project instructions explicitly say
that origin serves both surfaces.
