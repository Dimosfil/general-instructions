# Collect Reusable Project Experience For GI

## Purpose

Teach consuming projects to act as experience sources for `gi`. Projects should
capture reusable workflows, failure patterns, token-saving tactics, startup
retrieval improvements, and instruction improvements as reviewable
recommendations for `general-instructions`.

## Source Files

- `AGENTS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/project-memory-README.template.md`
- `patterns/AGENT_EXPERIENCE_SQLITE.md`
- `GENERAL_DEVELOPMENT_PLAYBOOK.md`
- `COMMANDS.md`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/README.md`
- optional local agent-memory schema or scripts
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Add or update local guidance saying the project is an experience source for
  `gi`.
- Ask agents to capture reusable workflows, repeated failure patterns,
  token-saving tactics, startup retrieval improvements, and instruction
  improvements that could help other projects.
- Store recommendations in the shared `general-instructions/updates/` folder
  when available; otherwise use a local fallback such as
  `tools/project-memory/instruction-updates/`.
- Recommendations should include the observed problem, proposed reusable rule or
  artifact, evidence paths or commands, expected benefit, risks, and privacy
  review.
- Do not include secrets, credentials, private user data, production data, or
  unnecessary project-specific details.
- Treat recommendations as intake only. Do not mark them as accepted guidance
  until they are reviewed and moved into shared instructions, patterns,
  templates, checklists, or migrations.
- Update `tools/project-memory/instruction-kit.json` after successful file
  changes, not before.

## Verification

- Reread changed local instruction and project-memory files.
- Confirm recommendations are described as intake only, not accepted rules.
- Confirm no guidance tells consuming projects to read the shared library's
  `updates/` during startup or bootstrap.
- Confirm privacy restrictions are present.
- Run `git diff --check` when the project is a git repository.

## Rollback Or Conflict Notes

If the project already has a different improvement-intake workflow, preserve the
local workflow and add only the reusable `gi` experience-source intent.
