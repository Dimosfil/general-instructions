# Add Answer-First Response Style

Migration ID: `2026.07.08.1__add_answer_first_response_style`

## Summary

Require final answers and direct user-facing explanations to begin with the
concrete answer, decision, main conclusion, or requested status before caveats
and supporting detail.

## Instructions

- Start final answers and direct user-facing explanations with the concrete
  answer or decision whenever the user asked a question or needs an outcome.
- Lead with `Yes`, `No`, `Exactly`, `Not yet`, the main conclusion, or the
  requested status before caveats, evidence, context, or implementation
  details.
- Put nuance and supporting detail after the direct answer so the user can
  decide whether to keep reading.
- Keep the rule project-agnostic. Do not copy a concrete investigation example
  into shared instructions.
