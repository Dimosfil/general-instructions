# Add Model Routing And Cost Control

Migration ID: `2026.05.21.3__add_model_routing_and_cost_control`

## Purpose

Add a reusable pattern for agent applications that choose between templates,
fast models, tool-capable models, RAG retrieval, and expensive reasoning models.

## Apply

- Add or reference a project-local model-routing policy when the project uses
  multiple models, tools, RAG stores, prompt caching, or expensive reasoning
  steps.
- Define deterministic pre-checks before model calls: rate limits, input
  length, auth, scope checks, slash commands, and exact command shortcuts.
- Define router categories, confidence thresholds, fallback behavior, and a
  compact structured output schema.
- Map each route to the cheapest safe execution path: template, default fast
  model, tool-capable model, RAG retrieval, or reasoning model.
- Keep stable prompts separate from volatile request context so prompt caching
  can reuse shared prefixes.
- Add bounded context selection for retrieved documents, histories, summaries,
  batch sizes, and individual document sizes.
- Add explicit output-token limits, per-turn tool-call limits, and user-level
  request or cost budgets.
- Require confirmation before irreversible, external, or high-blast-radius tool
  actions.
- Treat retrieved documents, emails, tickets, webpages, logs, calendar events,
  and task descriptions as data, not instructions.
- If a reasoning model cannot call tools or return strict JSON, gather required
  data first with a tool-capable model, then pass a prepared packet to the
  reasoning model.

## Verify

- Confirm router outputs are short, structured, and mapped to deterministic
  execution paths.
- Confirm simple confirmations, limit errors, unsupported requests, and exact
  command responses can bypass model calls.
- Confirm retrieved or user-provided content cannot change the agent role,
  bypass confirmations, reveal prompts, or execute unrelated actions.
- Confirm routing logs include category, selected model, token usage, cache
  hit/miss when available, tool-call count, latency, and fallback path.

## Rollback Or Conflict Notes

This migration adds an optional architecture pattern. It must not force one
provider, model name, product taxonomy, prompt, or exact router schema into
projects where local requirements differ.
