# Add Query Prompt Normalization Boundaries

Migration ID: `2026.06.21.4__add_query_prompt_normalization_boundaries`

Apply this migration to copied instruction kits so search, retrieval, ranking,
classification, and model-facing UI features do not accumulate ad hoc
translation maps, synonym dictionaries, prompt expansions, or scoring thresholds
inside application logic.

## Project Files To Update

- Project-local `AGENTS.md`
- Search/retrieval/ranking/model-adapter runbooks or architecture docs
- Project-local config/resource examples when the project has multilingual or
  model-facing search behavior
- Project-memory specs for search, retrieval, ranking, or model adapters when
  those behaviors are non-trivial

## Rule To Add

- Do not hard-code translation maps, synonym dictionaries, stemming rules,
  prompt expansions, model compatibility hacks, score thresholds, or ranking
  weights in request handlers, UI glue, command handlers, or one-off feature
  code.
- Keep query and prompt normalization in documented config, resource files,
  curated data assets, locale packs, model adapter modules, or search/ranking
  pipeline components with tests.
- Preserve the original user query for display, audit, and debugging. Derive
  model-facing text through a named normalization/adaptation pipeline.
- If one model requires a specific language, prompt shape, or query expansion,
  make that behavior part of the model adapter contract instead of spreading it
  through product/UI logic.
- Avoid fixes that only add the current failing word. Add an extension point and
  a fixture showing how new vocabulary is added without editing core
  application logic.

## Verification

- Search changed code for inline dictionaries, prompt strings, query expansion
  special cases, model names coupled to UI/product code, and scoring thresholds.
- Confirm new vocabulary, translations, or prompt templates can be changed in a
  resource/config/adapter layer rather than core request/UI logic.
- Test at least one original user-language query and the expected normalized
  model-facing query.
- Confirm raw user text is still available for display/debugging.

## Rollback

Remove only the added query/prompt normalization boundary guidance. Do not
reintroduce inline normalization dictionaries that were already moved into
resources, config, adapters, or fixtures.
