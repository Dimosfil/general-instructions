# Keep checkboxes and labels on one line

Accepted version: `2026.06.18.7`

## Purpose

Prevent chat clients from rendering numbered checklist prompts as detached
checkbox rows followed by separate numbered label rows.

## Shared Source Files

- `AGENTS.md`
- `CHANGELOG.md`
- `VERSION.md`
- `patterns/GIT_WORKFLOW.md`
- `skills/task-manager-plans/SKILL.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`

## Project Files

- Project-local `AGENTS.md`
- Project-local working agreements or runbooks that render language,
  commit-language, project-language, or task-manager adapter checklists

## Problem

Some prompts can accidentally put the task-list marker on one Markdown line and
the numeric label on the next line, for example:

```markdown
- [x]
1. English
```

That renders as an isolated checkbox with the label visually detached below it.

## Apply

- Render every numbered checklist option as a plain inline checkbox marker on
  one physical Markdown line.
- Keep the checkbox marker, number, and label together:

```markdown
[x] 1. English
[x] 2. Russian
[ ] 3. Spanish
[ ] 4. German
[ ] 5. French
```

- Do not use Markdown task-list syntax such as `- [x] 1. English`.
- Do not emit a standalone checkbox line followed by a separate numbered label
  line.
- Keep accepting numeric replies against the visible numbers in the most recent
  checklist.

## Verification

- Show a language or task-manager adapter checklist in chat.
- Confirm every checkbox, number, and label renders on the same visual row.
- Run `git diff --check`.
