# Add Unified Project Language

## Purpose

Let users choose one ordered project language sequence for agent communication,
agent-created tasks, plans, checklists, task-manager updates, and commit-message
preferences.

## Merge Rules

- Treat `gi language`, `gi язык`, `ги язык`, `gi project language`,
  `gi проект язык`, `ги проект язык`, `gi язык проекта`, and `ги язык проекта`
  as the unified project-language command family.
- The unified project-language command updates both
  `tools/project-memory/system-preferences.json` and
  `tools/project-memory/git-preferences.json`.
- Apply the configured project or system language to agent-created task titles,
  task descriptions, task-manager updates, plans, checklists, progress updates,
  final answers, clarifying questions, and user-facing explanations.
- Do not use the configured language to rewrite existing task text, code,
  commands, logs, quoted text, or a response language explicitly requested for a
  specific message.
- Preserve the user's selected language order. The first selected language is
  primary for agent-created tasks and user-facing communication; commit-message
  preferences use the same ordered sequence.
- Preserve existing separate `gi commit language` and `gi system language`
  command behavior for compatibility.
