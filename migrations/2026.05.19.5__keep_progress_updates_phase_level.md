# Keep Progress Updates Phase Level

## Purpose

Prevent agents from filling the chat with command-by-command narration during
debugging, implementation, and verification work.

## Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Add progress guidance that updates should be phase-level, not command-level.
- Agents should not:
  - narrate after every command batch;
  - report counters such as "ran 4 commands" or "completed 5 commands";
  - live-blog each intermediate hypothesis during normal investigation.
- Agents should update the user when:
  - the work phase changes;
  - a meaningful finding changes the next step;
  - a blocker appears;
  - work has been quiet long enough that the user needs reassurance.
- Routine observations, command names, and detailed logs should be batched
  internally and summarized in the final response or surfaced for failure
  diagnosis.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This rule should not prevent brief user-facing updates during long-running work.
It only suppresses repetitive command-by-command narration and noisy
intermediate speculation.
