# Optimize Total Live Context

## Purpose

Reduce context growth by focusing on total live context rather than treating
`cached input` as the primary optimization target.

## Source Files

- `AGENTS.md`
- `patterns/RAG_STARTUP_FLOW.md`
- `templates/AGENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat `cached input` as a symptom, not the main optimization target.
- Optimize for smaller total live context: current input plus cached context.
- Start a new session for unrelated new tasks when old context is no longer
  useful.
- Prefer compact handoff summaries over carrying long investigation history
  forward.
- Read only task-relevant file slices, targeted search results, or small line
  ranges instead of full large files.
- Split multi-step R&D into separate tasks when later steps do not need the full
  previous reasoning trace.
- Keep progress updates phase-level and concise so successful work does not add
  avoidable chat bulk.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration tightens token-economy behavior only. It does not prohibit longer
sessions, broad reads, or deeper investigation when they are explicitly requested
or required for incident debugging, release checks, or unresolved failures after
targeted searches.
