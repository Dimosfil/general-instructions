# Resolve FTP Services Through Config Service

Migration ID: `2026.05.29.3__resolve_ftp_services_through_config_service`

## Purpose

Let projects discover shared FTP/FTPS/SFTP services through config-service and
add a manual command for registering or selecting an FTP service record.

## Apply

- Treat `gi ftp service`, `gi ftp сервис`, and `ги фтп сервис` as requests to
  manually register, inspect, or select an FTP/FTPS/SFTP service record in
  config-service without uploading.
- When a project needs FTP and no local `serviceId` is selected, query
  config-service for FTP-capable services before asking the user for host
  details.
- If exactly one matching FTP service exists, read and verify its contract, then
  use it.
- If several matching FTP services exist, ask the user to choose with the same
  numbered Markdown checkbox style used by language selection, and accept
  numeric replies against that latest checklist.
- Store only non-secret discovery metadata and secret reference names in
  config-service. Never store raw passwords, tokens, private keys, or private
  remote paths there.
- Keep project-specific upload settings, such as `localPath`, `remotePath`, and
  `cleanRemote`, in the project-local FTP config unless local policy defines a
  stricter split.

## Verification

- Run `git diff --check`.
- Confirm `gi ftp service` does not upload files.
- Confirm FTP deploy setup checks config-service before asking for host details
  when no local FTP `serviceId` is selected.
- Confirm multiple matching services produce a numbered Markdown checkbox
  choice list.
- Confirm secrets are not copied into config-service records, logs, or final
  responses.

## Rollback Or Conflict Notes

If a project already has a stricter deploy-service registry, preserve the local
contract and map `gi ftp service` to that existing registry workflow.
