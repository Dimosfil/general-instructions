# Number Commit Language Selection

Migration ID: `2026.05.18.3__number_commit_language_selection`

## Purpose

Make ambiguous commit-language prompts easier to answer by adding numbered
options and an explicit explanation that English is the required primary
commit-message language.

## Shared Source Files

- `COMMANDS.md`
- `patterns/GIT_WORKFLOW.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files

- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/instruction-kit.json`

## Apply

- When the user asks to choose commit-message languages without naming the
  languages, ask with a concise numbered Markdown checklist.
- Show `English` as selected, primary, required, and not removable.
- Mark current additional languages as checked.
- Tell the user they may reply with numbers or language names.

Example:

```markdown
Which additional commit-message languages should be enabled?

English is the required primary commit-message language and cannot be disabled.
Reply with numbers or language names for any additional languages to enable.

1. [x] English (primary, required)
2. [x] Russian
3. [ ] Spanish
4. [ ] German
5. [ ] French
```

## Verify

- Ask `gi выбрать язык` without naming languages.
- Confirm the response shows numbered options, keeps English selected, explains
  that English is required, and accepts either numbers or names.
