# Add gi full test task commands

## Summary

Add `gi test task` / `gi test` as commands for setting a release/full-system
verification workload and then running the current project's documented test
flow against that workload.

## Scope

- `AGENTS.md`
- `COMMANDS.md`
- `patterns/PROJECT_TESTING_STRATEGY.md`
- `patterns/AGENTS_RUNTIME/09-project-operation-commands.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`
- `VERSION.md`
- `CHANGELOG.md`

## Rules

- Treat `gi test task`, `gi testing task`, `gi тест таск`, `ги тест таск`, and
  equivalent wording as requests to set the active release/full-system
  verification workload for the current project.
- The supplied task text is the selected scenario for the next `gi test`; it is
  not evidence that the scenario already passed.
- Store or track the selected task through project-local instructions when they
  define a test-task location; otherwise keep it in current chat context and
  report where it is tracked.
- Treat `gi test`, `ги тест`, `gi full test`, `gi release test`,
  `gi system test`, and equivalent full-project test wording as requests to run
  the current project's documented verification flow against the active test
  task.
- Do not confuse `gi test` with `gi test plan`; `gi test plan` remains
  plan-only by default.
- Before running `gi test`, verify exact current commands, services, app set,
  ports, routes, payloads, environment variables, storage, auth, queues,
  workers, and health checks from project-local instructions, README,
  manifests, runbooks, test configs, and source entry points.
- Old handoff summaries, screenshots, completed demo artifacts, previous task
  statuses, and old chat snippets are evidence only. They do not satisfy a
  fresh `gi test` request; rerun current documented checks or report the exact
  blocker.

## Verification

- Confirm `COMMANDS.md` lists `gi test task` and `gi test`.
- Confirm runtime operation commands document the run-vs-plan distinction.
- Confirm `patterns/PROJECT_TESTING_STRATEGY.md` treats previous runs as
  evidence only.
- Run `git diff --check`.

## Rollback

Remove only the `gi test task` / `gi test` command guidance, migration entry,
and version/changelog metadata added by this migration.
