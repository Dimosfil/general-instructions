# Clarify Update Intake Project Ownership

## Purpose

Prevent shared-instruction maintenance from treating update recommendations as
permission to inspect source projects.

## Source Files

- `AGENTS.md`
- `GENERAL_DEVELOPMENT_PLAYBOOK.md`

## Project Files To Update

- `AGENTS.md`
- `tools/AGENT_WORKING_AGREEMENTS.md` if it contains update-intake guidance

## Merge Rules

- Treat recommendation source projects and owners as provenance only.
- Reading a recommendation in this repository's `updates/` folder is allowed
  during `general-instructions` maintenance.
- Evidence paths, project names, task-manager notes, product plans, or owner
  labels in a recommendation are not permission to read, search, edit, or
  inspect the source project.
- Ask the user or that project's owner for an explicit concrete path and action
  before crossing the repository boundary.
- Keep the existing rule that external projects must not read the shared
  library's `updates/` folder during startup or bootstrap.

## Verification

- Reread update-intake and scope-boundary rules.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration tightens cross-project boundaries only. It does not change the
format of update recommendations.
