# Analyze Bug Evidence Before Fixing

## Purpose

Prevent agents from immediately changing code when the user only shares
screenshots, logs, pasted errors, or other bug evidence.

## Source Files

- `AGENTS.md`
- `templates/AGENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat screenshots, logs, pasted errors, or other bug evidence as requests for
  analysis first.
- Explain the likely issue and ask what action the user wants before editing
  files.
- Proceed directly to fixing only when the user explicitly asks for a fix, such
  as `fix`, `почини`, or `gi почини`.
- Keep any analysis within the current project scope and existing privacy
  rules.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration changes default response posture for bug evidence only. It does
not block direct implementation requests or explicit repair commands.
