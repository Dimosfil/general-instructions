# Prompt For Task Manager After Update

Migration ID: `2026.05.18.6__prompt_for_task_manager_after_update`

## Purpose

Make task-manager setup visible immediately after an instruction-kit update adds
or updates task-manager plan sync.

## Shared Source Files

- `COMMANDS.md`
- `skills/task-manager-plans/SKILL.md`

## Project Files

- Optional: `tools/project-memory/task-managers.json`
- Optional: `skills/task-manager-plans/SKILL.md`
- Optional: `skills/task-manager-plans/references/managers/worknest.md`

## Apply

- After `gi обновить` / `gi обновись`, if task-manager plan sync became
  available and `tools/project-memory/task-managers.json` is missing or has no
  enabled managers, immediately ask the user to choose task managers.
- Use a concise numbered Markdown checklist with checkbox items, matching the
  commit-language selection style.
- Include available adapters from `skills/task-manager-plans/references/managers/`
  and a `none` option.
- Accept user replies by number or manager name.
- Do not enable WorkNest or any other manager automatically.

Example:

```markdown
Выбери task-manager adapters для plan sync:

1. [ ] WorkNest — отправлять планы в WorkNest raw intake.
2. [ ] none — ничего не подключать сейчас.
```

## Verify

- Confirm `gi обновить` does not silently finish task-manager setup when no
  manager is configured.
- Confirm `gi обновить` asks with numbered checkbox options.
- Confirm choosing `none` leaves task-manager sync disabled.
