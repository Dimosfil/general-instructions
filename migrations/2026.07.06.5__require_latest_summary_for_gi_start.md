# Require Latest Summary For GI Start

Migration ID: `2026.07.06.5__require_latest_summary_for_gi_start`

## Summary

Tighten `gi start` / `gi restore` so the newest handoff summary is read as the
primary continuation artifact for a new chat, not treated as optional metadata.

## Instructions

- For `gi start`, `gi restore`, and title-only startup restore, read the latest
  handoff summary enough to recover the current topic, key theses or decisions,
  blockers, and next useful direction.
- Do not count seeing only the summary filename, timestamp, or metadata as
  successful context restoration.
- Keep startup output compact: summarize the restored handoff context and ask
  what to do next.
- Keep token economy intact: do not read older summaries, full runbooks, logs,
  memory dumps, diffs, or unrelated notes unless a concrete task requires them.

## Rollback Or Conflict Notes

This does not require agents to paste or load an entire old chat into context.
The latest handoff summary exists to carry the condensed essence of that chat,
so startup restore should read the summary content while still avoiding broad
context restoration.
