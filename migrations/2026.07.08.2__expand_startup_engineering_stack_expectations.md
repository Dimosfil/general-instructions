## Instructions

- Treat C#/.NET async, cancellation, bounded concurrency, background services,
  and observable failure behavior as core engineering expectations.
- Treat LLM and RAG integration as business orchestration with explicit
  contracts around prompt assembly, retrieval, embeddings, model routing, tool
  calls, semantic cache, provider adapters, and cost controls.
- Treat PostgreSQL persistence as architecture-sensitive: inspect query shape,
  indexes, transaction boundaries, locking, connection pooling, migrations, and
  current access patterns before changing data behavior.
- Apply GoF, GRASP, SOLID, DRY, KISS, and clean-code principles by intent and
  proportionality. Avoid design-pattern ceremony that hides simple behavior or
  duplicates framework conventions.
- Treat GitLab or similar CI/CD edits as operational contract changes requiring
  current pipeline, runner, environment, secret, artifact, cache, deploy,
  rollback, and branch-protection context.
