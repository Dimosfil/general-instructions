# Use Checklist For Commit Language Selection

Migration ID: `2026.05.16.6__use_checklist_for_commit_language_selection`

## Purpose

Make ambiguous commit-message language selection prompts easier to answer by
using a concise Markdown checklist.

## Shared Source Files

- `patterns/GIT_WORKFLOW.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`
- `COMMANDS.md`

## Project Files

- `tools/AGENT_WORKING_AGREEMENTS.md`
- `tools/project-memory/git-preferences.json`
- `tools/project-memory/instruction-kit.json`

## Apply

- When the user asks to choose/select commit-message languages without naming
  languages, present a concise Markdown checklist instead of a prose-only list.
- Show `English` as selected and primary.
- Mark currently enabled additional languages as checked.
- Include available additional languages: `Russian`, `Spanish`, `German`, and
  `French`.
- Ask the user to reply with language names or numbers.

## Example

```markdown
Which additional commit-message languages should be enabled?

- [x] English (primary, always on)
- [x] Russian
- [ ] Spanish
- [ ] German
- [ ] French
```

## Verify

- Confirm `gi выбрать язык коммита` produces a Markdown checklist.
- Confirm the agent does not automatically change preferences until the user
  answers with language names or numbers.
