new features, bug fixes, and release checks.

## Apply

- Treat `gi тест-план` and `gi test plan` as shared instruction-kit commands.
- Treat `gi проверить фичу`, `gi feature check`, `gi план проверки`, and
  `gi test strategy` as equivalent aliases.
- In the current project root, inspect local instructions, runbooks, scripts,
  package files, CI configs, and test directories to identify project-specific
  verification options.
- Produce a compact test plan that covers fast checks, focused automated tests,
  integration or smoke checks, manual checks, regression areas, unverified
  areas, and confidence.
- By default, plan checks but do not run them automatically. Run checks only
  when the user asks or when the current implementation task already requires
  verification.
- For UI projects, follow local instructions about browser/screenshot use. If
  automatic visual inspection is not allowed, provide a manual checklist.
- When the user asks to save the plan, use the structure from
  `templates/FEATURE_TEST_PLAN.template.md` in a project-local memory file.

## Verify

- Ask `gi тест-план` in a project with known local test commands.
- Confirm the agent finds project-specific commands before recommending generic
  checks.
- Confirm the response distinguishes checked, not checked, regression risks, and
  optional broader checks.
- Confirm the agent does not run tests by default for the planning-only command.
