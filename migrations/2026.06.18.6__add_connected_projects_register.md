# Add Connected Projects Register

Accepted version: `2026.06.18.6`

## Rule

Projects that depend on, research, vendor, or regularly interact with external
repositories, cloned examples, service projects, libraries, documentation sites,
upstream tools, or sibling workspaces must keep a connected-projects register in
project memory, preferably:

```text
tools/project-memory/specs/integration-contracts/connected-projects.md
```

For each connected project, record:

- purpose and short description;
- business or architectural role in the current project;
- local folder path when it is an approved workspace dependency;
- canonical Git, package, documentation, dashboard, or issue-tracker URLs;
- service IDs, runtime endpoints, contract endpoints, or discovery records;
- owner, source of truth, update cadence, and version or branch policy;
- data exchanged, API contract, generated artifacts, and runtime dependencies;
- setup, sync, build, test, and update commands that are safe to run;
- privacy, secret, license, and cross-repository access boundaries;
- current status, caveats, and the reason the dependency still exists.

Agents must read this register before touching integrations, nested
repositories, cloned examples, or external project folders. Update it in the
same scoped change when adding, removing, replacing, relocating, or materially
changing the role of a connected project. The register is not permission to
inspect arbitrary external folders; project-local scope and explicit user
requests still govern filesystem access.

## Rationale

Connected projects often become part of a product's business logic and
architecture even when their source lives outside the current repository. A
future agent needs to know what each external dependency is, where its source of
truth lives, why the project uses it, and what boundaries apply before it can
make safe integration decisions.

## Propagation

Apply this rule to:

- live repository instructions;
- copied-project templates;
- project-memory specification guidance;
- project-memory README and setup checklist;
- connected-projects starter template;
- instruction-kit metadata.

If a consuming project has known connected projects, create or update
`tools/project-memory/specs/integration-contracts/connected-projects.md` using
the starter template. If it has no connected projects, no register is required
until one is introduced.
