# Add GI Mod Game Path Command

Migration ID: `2026.07.08.4__add_gi_mod_game_path_command`

## Summary

Add `gi mod` / `谐懈 屑芯写` and `gi mod path <game-install-path>` /
`谐懈 屑芯写 锌褍褌褜 <锌褍褌褜-懈谐褉褘>` for game modding projects that must remember the
selected local game install path separately from mod project, mod install, and
log folders.

## Instructions

- Treat `gi mod`, `gi mod path`, `gi game path`, `gi mod game path`,
  `谐懈 屑芯写`, `谐懈 屑芯写 锌褍褌褜`, `谐懈 锌褍褌褜 懈谐褉褘`, and equivalent wording as a game
  modding project command.
- Before mod build, install, or debug actions, distinguish the current mod
  project root, selected game install root, user/game documents mod folder, and
  logs or crash-report folder.
- Store the selected game install root only in project-local ignored
  configuration, preferably `tools/project-memory/game-modding.local.json`.
  Keep fields non-secret and evidence-oriented: `game_name`,
  `game_install_path`, `mod_install_path`, `logs_path`, `launcher`,
  `detected_from`, `verified_at`, and notes.
- Add or keep `tools/project-memory/game-modding.local.json` ignored. Do not
  commit machine-specific game paths to shared instructions, templates, source
  code defaults, migrations, or ordinary docs.
- If a path is supplied in the command, resolve it, verify it exists, and check
  for game-specific evidence such as an executable, launcher manifest, app
  manifest, modding SDK folder, data/content folder, or runbook match. If
  evidence is weak, record that explicitly.
- If no selected game path is recorded, search project-local instructions,
  README, runbooks, manifests, existing ignored config, and project memory
  first. Inspect launcher library metadata only when local policy and user
  scope allow it. Do not scan arbitrary user folders, sibling projects, or
  whole drives without explicit user scope.
- If the game path remains unknown, ask one concise question for the missing
  game install root and name the local save target, for example
  `tools/project-memory/game-modding.local.json`.
- Include response shapes for missing and newly supplied game paths in both
  neutral English and Russian when the project/user language calls for Russian.
