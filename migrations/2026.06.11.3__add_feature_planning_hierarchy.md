# Migration: Add Feature Planning Hierarchy

Version: `2026.06.11.3`

Apply this migration to copied instruction kits that need agents to preserve
feature intent, workflow, and implementation planning across sessions, sprints,
and task-manager handoffs.

## Changes

- Extend feature workflow guidance so non-trivial features record:
  - feature idea;
  - functional description;
  - runtime workflow contract;
  - implementation plan;
  - sprint breakdown;
  - task breakdown with definitions of done.
- Clarify that sprint and task lists do not replace the feature contract:
  tasks say what to change, while the contract says what behavior must remain
  true.
- Tell agents to read the relevant feature contract or plan before changing the
  feature and to preserve the agreed behavior unless the user explicitly changes
  the agreement.
- Update feature workflow templates to include implementation plan, sprint, and
  task sections.

## Suggested Project Rule

- For non-trivial features, keep a durable project-local feature document with
  the feature idea, functional description, workflow contract, implementation
  plan, sprint breakdown, tasks, definitions of done, and verification.
- Use one sprint for small features and multiple sprints for larger work.
- Every sprint should have a goal, scope, dependencies, exit criteria, and
  verification.
- Every task should trace back to the feature workflow or implementation plan.
- Before marking a task done, confirm the task did not break the agreed
  sequence, branch, loading behavior, background work, or user-visible promise.
