# Harden GI Init Bootstrap Entrypoint

Migration ID: `2026.07.10.1__harden_gi_init_bootstrap_entrypoint`

## Summary

Make GI bootstrap discoverable before an agent can misclassify a canonical
repository link as a Git project-replacement or remote-management request, and
provide a deterministic installer for the copied instruction kit.

## Accepted Guidance

- Treat `инит [Dimosfil/general-instructions.git](https://github.com/Dimosfil/general-instructions.git)`,
  the plain canonical URL, the short repository form, and a local checkout as
  equivalent shared-instruction bootstrap sources.
- Resolve a Markdown link target before classifying the request.
- Use the current active project root as the default target. Do not require or
  persist a machine-specific drive or checkout path as the canonical source.
- Read root `BOOTSTRAP.md` and `patterns/SHARED_INSTRUCTIONS_BOOTSTRAP.md` before
  proposing Git or project operations.
- Never reinterpret the request as ordinary `git init`, project replacement,
  remote replacement/addition, a dependency, submodule, symlink, or live
  runtime reference.
- Prefer `tools/install-instruction-kit.ps1` for a deterministic fresh-project
  bootstrap. Preserve existing files and merge project-owned instructions
  deliberately when they already exist.
