# Add Task Manager Post Plan Command

Migration ID: `2026.05.18.7__add_task_manager_post_plan_command`

## Purpose

Add short commands for sending the current work plan to a configured task
manager and make manager setup collect required connection fields immediately.

## Apply

- Add `gi план` and `gi post plan` as short task-manager plan sending commands.
- Treat `gi отправь план`, `gi post work plan`, `gi send plan`, and
  `gi task plan` as aliases.
- On `gi план` / `gi post plan`, read the configured task managers and send the
  current plan to each enabled manager.
- If no plan is present in the user's message, use the current active plan from
  conversation or project memory. If no active plan exists, ask what plan to
  send.
- If no manager is configured, run the same checkbox selection flow as `gi tm`.
- If WorkNest is selected or enabled, require `base_url` before completing setup
  or sending. Read it from project instructions, environment/config, or ask the
  user. Do not leave `base_url: "TODO"`.
- For WorkNest, send through `POST /agent-intake/raw` and treat the response as
  an intake receipt, not a created-card confirmation.

## Verify

- Confirm `gi план` and `gi post plan` trigger task-manager plan sending.
- Confirm WorkNest setup asks for `base_url` when it is missing.
- Confirm `tools/project-memory/task-managers.json` contains no secrets.
