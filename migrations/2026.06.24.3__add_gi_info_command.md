## Merge Rules

- Add `gi info` and `–≥–ł –ł–Ĺ—Ą–ĺ` to the command index and compact help table.
- Treat the command as a documentation/inventory command, not a runtime start,
  test, install, or rebuild command.
- Require agents to read project-local instructions, README, docs, runbooks,
  existing project-memory specs, and the canonical stack inventory before broad

- Require evidence-backed verification against manifests, config, run
  instructions, source entry points, and tests where available.
- Keep purpose, visible functionality, workflows, operations, and stack pointers

  `tools/project-memory/specs/technology-stack.md` unless local instructions

  invariants, and architecture decisions in project memory, linking to them from

  missing-source notes instead of guessing.
- Do not install dependencies, start services, rebuild indexes, call external
  APIs, read secrets, or inspect private paths outside the project root for this
  command unless the user explicitly approves that scope.
