# Add GI Help Command Index

## Purpose

Propagate the accepted rule that `gi help` / `ги хелп` and command-list aliases
show a compact list of available GI chat commands with short descriptions.

This gives agents and users one lightweight entry point for discovering
commands without running startup restore, task-manager calls, or old work.

## Source Files

- `AGENTS.md`
- `COMMANDS.md`
- `templates/AGENTS.template.md`
- `templates/AGENT_WORKING_AGREEMENTS.template.md`

## Project Files To Update

- `AGENTS.md`
- `COMMANDS.md` when present
- `tools/AGENT_WORKING_AGREEMENTS.md` when present

## Merge Rules

- Add `gi help`, `gi хелп`, `ги help`, `ги хелп`, `gi commands`,
  `gi команды`, and `ги команды` as informational GI chat commands.
- Add or update a local command index with command names and one short
  description per command. Prefer project-local command additions over the
  shared baseline.
- Keep help read-only. It must not run startup restore, resume old work, call
  services or task managers, mutate files, execute listed commands, or ask the
  user to choose a command unless the user asks for help with a specific one.
- If the project has no `COMMANDS.md`, use the local `AGENTS.md`, runbook, or
  working-agreements file as the command source and consider adding a compact
  `COMMANDS.md` only if local policy allows it.

## Verification

- Confirm `gi help` aliases are documented in project instructions.
- Confirm the command index includes short descriptions for all known GI
  commands in the project.
- Confirm help behavior is read-only and does not trigger startup restore or
  task execution.

## Rollback

Remove only the help aliases and command-index guidance. Leave existing command
rules intact.
