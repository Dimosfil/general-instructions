# Restart Runtime After Feature Changes

## Purpose

Prevent verification against stale frontend assets, backend processes, API
routes, schemas, or cached responses after feature work.

## Source Files

- `AGENTS.md`
- `patterns/PROJECT_TESTING_STRATEGY.md`
- `templates/AGENTS.template.md`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- After implementing a frontend, backend, API, or full-stack feature, restart
  the affected dev server or backend process when local run instructions provide
  a restart command or hot reload is uncertain.
- Refresh the browser, client, or API caller before verification.
- Probe changed API endpoints or route contracts after restart when they feed
  the UI.
- Do not assume updated HTML or JavaScript means the backend process has loaded
  matching code.
- Mention any restart or refresh that was skipped and why.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration should follow project-local run instructions. It does not require
destructive process management, broad service restarts, or browser automation
when local policy says the user will inspect UI manually.
