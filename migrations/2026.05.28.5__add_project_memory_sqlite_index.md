# Add Project Memory SQLite Index

Migration ID: `2026.05.28.5__add_project_memory_sqlite_index`

## Purpose

Provide a small, rebuildable local SQLite search index for project memory and
repository text without committing generated database files.

## Apply

- Add `tools/project-memory/README.md` to document durable memory files and the
  generated SQLite index.
- Add `tools/project-memory/build_project_memory_index.py` as a stdlib-only
  helper that can rebuild, query, and export notes from a local SQLite database.
- Ignore generated SQLite database files such as
  `tools/project-memory/*.sqlite`, `*.sqlite-*`, `*.db`, and `*.db-*`.
- Keep long-lived memory reviewable in Markdown or JSON. Treat SQLite as a
  generated local search index that can be rebuilt from git tracked files.

## Verification

- Run `python .\tools\project-memory\build_project_memory_index.py rebuild`.
- Run `python .\tools\project-memory\build_project_memory_index.py stats`.
- Confirm the generated database remains ignored by git.
- Run `git diff --check`.

## Rollback Or Conflict Notes

If a project already has a project-memory indexer, keep the local project tool
and adapt only the ignore rules or README guidance that still apply.
