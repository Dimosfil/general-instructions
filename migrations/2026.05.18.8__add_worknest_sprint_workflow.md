# Add WorkNest Sprint Workflow

Migration ID: `2026.05.18.8__add_worknest_sprint_workflow`

## Purpose

Add WorkNest-specific guidance for turning accepted plans into dated Markdown
sprint folders with ordered task files.

## Shared Source Files

- `skills/task-manager-plans/references/managers/worknest.md`

## Project Files

- Optional: `skills/task-manager-plans/references/managers/worknest.md`

## Apply

- When a WorkNest-style Markdown project receives an accepted plan, treat it as
  a sprint.
- Create a unique dated sprint folder directly inside the target project:
  `projects/<project-id>/YYYY-MM-DD_<sprint-slug>/`.
- Do not create a shared `sprints/` container by default.
- Add `sprint.md` with sprint metadata, source plan/request, goal, and ordered
  task list.
- Split the plan into numbered task files such as
  `001_short-task-title.md`, `002_short-task-title.md`, and
  `003_short-task-title.md`.
- Include status, sprint id, order, project id, agent, source plan id,
  dependencies, and tags in each task file.
- Execute task files by ascending `Order`; the next task is the first task whose
  status is `todo` or `ready`.
- On completion, mark the task file `done` and append a concise result section.
- Do not block the initial sprint workflow on a full testing model.

## Verify

- Confirm WorkNest adapter guidance includes the sprint folder structure.
- Confirm the workflow is scoped to WorkNest-style Markdown projects, not all
  task managers.
