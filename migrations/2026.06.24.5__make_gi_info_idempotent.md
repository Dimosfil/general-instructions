# Make gi info idempotent

## Purpose

Clarify that `gi info` / `ги инфо` updates project information only when
verified facts are missing or stale. It should not rewrite documentation on
every run.

## Files

- `COMMANDS.md`
- `patterns/AGENTS_RUNTIME/01-purpose.md`
- `patterns/AGENTS_RUNTIME/06-tool-usage-and-token-economy.md`
- `patterns/PROJECT_DOCUMENTATION_LAYERS.md`
- `CHANGELOG.md`
- `VERSION.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Before editing, compare verified current facts with the existing project
  overview and canonical stack inventory.
- If purpose, user-visible functionality, common workflows, commands,
  operations, troubleshooting pointers, and stack facts already match current
  evidence, report that project information is current and do not rewrite files.
- If only part of the inventory changed, update only the affected sections.
- Preserve unchanged sections, existing links, and canonical source-of-truth
  boundaries.
- Avoid whole-file rewrites, broad reformatting, wording churn, and
  retranslating unchanged content just because `gi info` was invoked.
- Continue marking missing or contradictory facts as gaps/TODOs with evidence
  paths or missing-source notes.
