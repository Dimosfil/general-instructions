# Log User-Reported Agent Bugs And Tools Artifact Boundary

## Summary

Start a maintenance-only log for recurring user-reported agent-rule failures and
make the `tools/` boundary explicit: durable tooling belongs there, while
generated product outputs and selected-run artifacts do not.

## Instructions

- When the user reports an agent-rule failure, repeated behavior bug, or asks to
  keep a log of such bugs, append a compact entry to
  `updates/USER_REPORTED_AGENT_BUG_LOG.md` in this repository when available.
- Keep that log maintenance-only. Consuming projects must not read it during
  startup or bootstrap.
- Record the date, symptom, likely violated rule or rule gap, evidence summary,
  privacy review, status, and accepted migration or follow-up.
- Use `tools/` for durable development tooling, automation scripts, adapters,
  bootstrap commands, deployment helpers, and redacted example manifests.
- Do not use `tools/` as the default destination for generated product output,
  selected-run artifacts, uploaded site contents, screenshots, raw exports,
  build bundles, downloaded datasets, or one-off work results.
- Store those outputs in documented project-local artifact, evidence, output,
  data, docs-asset, build, or release locations. Keep only compact manifests,
  checksums, summaries, or links in tooling or project memory when needed.

## Verify

- Confirm `updates/USER_REPORTED_AGENT_BUG_LOG.md` exists and has the reported
  incident entry.
- Confirm `patterns/AGENTS_RUNTIME/14-update-intake.md` documents when to append
  to the log.
- Confirm `patterns/AGENTS_RUNTIME/04-content-and-authoring.md` and
  `patterns/DEVELOPMENT_TOOL_PRODUCT_BOUNDARIES.md` state the `tools/` artifact
  boundary.
- Confirm `AGENTS.md`, `GENERAL_DEVELOPMENT_PLAYBOOK.md`,
  `updates/README.md`, and `INDEX.md` mention the new boundary or log where
  relevant.
- Confirm `templates/AGENTS.template.md` and
  `templates/project-memory-README.template.md` carry the copied-project
  guidance.

## Rollback Or Conflict Notes

This rule does not ban deployment scripts, bootstrap helpers, redacted examples,
or small reviewable manifests from `tools/`. It bans using `tools/` as an
implicit storage area for product outputs and one-off generated results.
