# Add GI Git Summary Command

Migration ID: `2026.05.18.1__add_gi_git_summary_command`

## Purpose

Add a short `gi` command for summarizing the latest git commit in the current
project without dumping a full diff or changing files.

- `COMMANDS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`

## Apply

- Treat `gi гит-обзор` and `gi git summary` as shared instruction-kit commands.
- Treat `gi гит обновления`, `gi гит-обновления`, and `gi обзор коммита` as
  equivalent aliases when the user's intent is to inspect the latest git commit.
- Run the command in the current project root. Do not switch to another
  repository unless the user explicitly asks.
- If the current project is not a git repository, report that clearly and stop.
- By default, summarize the latest commit in the current branch history by
  commit date. Use all branches or another revision only when the user asks.
- Gather compact information with commands such as:

```powershell
git log -1 --date=iso-strict --format=fuller
git show --stat --name-status --find-renames --find-copies --format=fuller HEAD
```

- Report the commit hash, date, author, subject/body, changed files, compact
  stats, inferred purpose, and notable risks or checks.
- Do not print the full diff by default.
- Do not create a handoff summary file, commit, or push for this command.

## Verify

- In a git repository, ask `gi гит-обзор`.
- Confirm the agent reports the latest commit metadata, changed files, stats,
  inferred purpose, and risks without printing a full diff.
- In a non-git folder, confirm the agent reports that git metadata is unavailable
  instead of treating the command as an instruction-kit update.
