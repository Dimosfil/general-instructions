# Continue Development After Chat Secret Exposure

Migration ID: `2026.07.15.1__continue_development_after_chat_secret_exposure`

## Purpose

Prevent a credential pasted into chat from blocking an entire development task
while preserving safe secret handling and rotation guidance.

## Accepted Guidance

- Treat a credential pasted into chat as compromised and recommend rotation,
  but do not require rotation before unrelated safe development continues.
- Warn once without repeating the secret and do not repeatedly interrupt the
  user about the same exposure.
- Continue local implementation, configuration wiring, mocked or sandboxed
  tests, documentation, and every other independent step that does not expose
  or unsafely persist the credential.
- Use only approved non-echoing secret paths for authenticated operations. When
  none exists, mark only the affected operation blocked or unverified and
  continue the remaining work.
- Never copy the exposed value into source, committed config, logs, project
  memory, task-manager payloads, generated artifacts, or later responses.
