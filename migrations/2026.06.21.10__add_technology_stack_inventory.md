# Add Technology Stack Inventory

## Purpose

Make the current technology stack a durable project-memory source of truth for
GI-enabled projects.

## Source Files

- `patterns/TECHNOLOGY_STACK_INVENTORY.md`
- `templates/TECHNOLOGY_STACK.template.md`
- `templates/project-memory-README.template.md`
- `templates/instruction-kit.template.json`
- `AGENTS.md`
- `GENERAL_DEVELOPMENT_PLAYBOOK.md`
- `INDEX.md`
- `VERSION.md`
- `CHANGELOG.md`

## Project Files

- `tools/project-memory/specs/technology-stack.md`
- `tools/project-memory/README.md`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Create `tools/project-memory/specs/technology-stack.md` from the template when
  it does not exist.
- If the project already has a stack ADR or architecture note, keep it and link
  it from `technology-stack.md`, or migrate the current facts without losing the
  decision history.
- Populate stack facts from current manifests, runbooks, project-local
  instructions, and config files. Mark unknown or inferred entries explicitly.
- Do not store secrets, credentials, private data, or machine-specific absolute
  paths unless the project explicitly records an approved external workspace
  dependency.
- Update `tools/project-memory/instruction-kit.json` only after the file changes
  have been applied and checked.

## Verification

- Confirm `tools/project-memory/specs/technology-stack.md` exists for the
  GI-enabled project.
- Confirm any recorded commands and stack facts match current manifests or
  runbooks.
- Validate edited JSON instruction-kit metadata.
- Run `git diff --check` or the project equivalent when the project is a git
  repository.

## Rollback

Remove only the newly created `technology-stack.md` file and metadata entry if
the migration must be reverted. Do not delete pre-existing ADRs or architecture
notes.
