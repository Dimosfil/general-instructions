project-configured task managers.

## Apply

- Ask the user which task managers to connect before enabling this skill in a
  new project. Offer the available adapters and a `none` option as a numbered
  Markdown checklist with checkbox items.
- Copy the `task-manager-plans` skill only when the project needs task-manager
  plan sync.
- Create `tools/project-memory/task-managers.json` from the template when the
  user connects at least one manager.
- For WorkNest, use the raw intake MVP and read the API URL from the project
  task-manager config.
- Keep secrets out of task-manager config. Store credentials only in approved
  secret stores, environment variables, or configured connectors.
- For `gi tm`, check whether managers are already configured. If yes, update the
  copied task-manager skill/config from the shared kit when needed. If no, ask
  the user to choose managers with a numbered Markdown checkbox checklist.

## Verify

- Confirm `tools/project-memory/task-managers.json` contains no secrets.
- Confirm selected manager adapters exist under `skills/task-manager-plans/`.
- Confirm `tools/project-memory/instruction-kit.json` records this migration
  after applying it.
