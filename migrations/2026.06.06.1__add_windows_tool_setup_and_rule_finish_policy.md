# Migration: Add Windows Tool Setup And Rule-Finish Policy

Version: `2026.06.06.1`

Apply this migration to copied instruction kits that need the latest Windows
tooling and shared-instruction-library maintenance rules.

## Changes

- Add Windows command guidance to the project `AGENTS.md`:
  - prefer `Invoke-RestMethod` and `Invoke-WebRequest` instead of `curl.exe`;
  - avoid probing `curl.exe` with `where.exe curl` or `Get-Command curl` unless
    the user explicitly asks for curl diagnostics;
  - prefer trusted helper binaries from `C:\Users\<user>\.codex\bin` before
    WindowsApps or System32 shims;
  - when antivirus blocks agent commands, trust narrow Codex-owned folders such
    as `.codex\.sandbox-bin\` and `.codex\bin\`, not broad System32 or
    PowerShell exclusions.
- In shared instruction-library projects, add a finish rule: a user request to
  add or accept a reusable rule is approval to verify, commit, and push only the
  scoped rule changes, then run the `gi обновить` update flow when accepted
  instruction-kit propagation applies.
- Add the Windows setup checklist item to any local new-project agent setup
  checklist if the project keeps one.

## Verification

- Run `git diff --check`.
- If updating JSON instruction-kit metadata, validate it with `ConvertFrom-Json`
  or another JSON parser.
- Confirm no unrelated dirty worktree changes, secrets, logs, local databases,
  or generated caches are included in the commit.
