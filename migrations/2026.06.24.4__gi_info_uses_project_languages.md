# Make gi info use project languages

## Purpose

Clarify that `gi info` / `ги инфо` writes collected project information in the
configured project working-environment languages selected by `gi язык` /
`gi language`.

## Files

- `COMMANDS.md`
- `patterns/AGENTS_RUNTIME/01-purpose.md`
- `patterns/AGENTS_RUNTIME/06-tool-usage-and-token-economy.md`
- `patterns/AGENTS_RUNTIME/11-language-preferences.md`
- `patterns/PROJECT_DOCUMENTATION_LAYERS.md`
- `CHANGELOG.md`
- `VERSION.md`
- `templates/instruction-kit.template.json`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- When `gi info` creates or updates project documentation, use
  `tools/project-memory/system-preferences.json` project working-environment
  languages, preserving the selected order from `gi язык` / `gi language`.
- Treat the first configured project working-environment language as primary.
- If one language is configured, write only that language.
- If multiple languages are configured, write the primary-language text first
  and add one clear translation per additional configured language.
- Do not use commit-message language preferences or task-manager language
  preferences for `gi info` project documentation.
- Continue preserving existing task text, code, commands, logs, quoted text, and
  explicitly user-requested response language exceptions.

## Verification

- Confirm `COMMANDS.md`, runtime modules, and documentation-layer guidance

- Confirm `VERSION.md`, changelog, migration metadata, and templates include

## Rollback

Remove only the language clarification added for `gi info`, the migration
entry, changelog note, version bump, and metadata entries added by this
migration.
