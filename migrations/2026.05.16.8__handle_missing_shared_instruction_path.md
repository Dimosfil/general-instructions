# Handle Missing Shared Instruction Path

Migration ID: `2026.05.16.8__handle_missing_shared_instruction_path`

## Purpose

Keep instruction-kit update checks useful when a copied project records a shared
source checkout/cache that is unavailable in the current environment.

## Shared Source Files

- `templates/check-instruction-kit-updates.template.ps1`
- `patterns/INSTRUCTION_KIT_MIGRATIONS.md`
- `COMMANDS.md`

## Project Files

- `tools/check-instruction-kit-updates.ps1`
- `tools/project-memory/instruction-kit.json`

## Apply

- Update `tools/check-instruction-kit-updates.ps1` from the template or merge
  the path-resolution behavior.
- Accept an optional `-SharedLibraryPath` parameter for local checkout/cache
  overrides.
- Prefer the configured source repo when no usable checkout/cache exists.
- Treat unavailable saved paths as recoverable. Do not emit a raw missing-drive
  failure as the final answer.
- If no usable source exists, ask the user for the shared instruction source.

## Verify

- Run `tools/check-instruction-kit-updates.ps1` in an environment where the
  recorded shared path is unavailable and confirm it exits with a clear message.
- Run it again with a configured source repo or explicit checkout/cache and
  confirm it reads `VERSION.md` from that usable source.
