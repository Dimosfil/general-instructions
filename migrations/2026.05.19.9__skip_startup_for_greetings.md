# Skip Startup For Greetings

## Purpose

Prevent trivial first messages, greetings, thanks, and acknowledgements from
loading project context before the user gives an actual task.

## Source Files

- `AGENTS.md`
- `patterns/FIRST_MESSAGE_HANDLING.md`
- `patterns/RAG_STARTUP_FLOW.md`
- `templates/AGENTS.template.md`
- `templates/agent-start.template.ps1`
- `templates/instruction-kit.template.json`

## Project Files To Update

- `AGENTS.md`
- `tools/agent-start.ps1`
- `tools/project-memory/instruction-kit.json`

## Merge Rules

- Treat short greetings, thanks, acknowledgements, and status-neutral messages
  as no-ops unless they include an explicit task, path, command, error, or
  project question.
- Do not run startup restore, read files, inspect summaries, query memory, check
  logs, or inspect git state for no-op messages.
- Reply briefly and ask what the user wants to do next.
- If the user then gives a task, restore only the context needed for that task.
- For `gi start`, `gi restore`, and title-only first messages, restore only the
  minimum orientation needed for the next turn: local instructions, latest
  summary metadata or relevant sections, and compact git state.
- Do not read full summaries, runbooks, memory notes, logs, or diffs during
  compact restore unless a concrete task needs them.
- Reduce default startup-script output limits so copied kits do not print large
  instruction or summary blocks by default.

## Verification

- Reread edited instruction files.
- Validate `tools/project-memory/instruction-kit.json` if it is updated.
- Run `git diff --check`.

## Rollback Or Conflict Notes

This migration changes only default startup behavior. It does not block explicit
startup commands, project-study requests, debugging work, or user-directed file
inspection when the user gives a concrete task.
