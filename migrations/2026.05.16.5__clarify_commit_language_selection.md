# Clarify Commit Language Selection

Migration ID: `2026.05.16.5__clarify_commit_language_selection`

## Purpose

Prevent agents from guessing commit-message language preferences from the user's
UI language or message language, and add a short `gi` command prefix for shared
instruction-kit commands.

- `patterns/GIT_WORKFLOW.md`

- `tools/project-memory/instruction-kit.json`

## Apply

- Keep English as the primary commit-message language.
- Treat short chat commands that start with `gi` as shared instruction-kit
  commands for `general-instructions`, not as product work for the current
  project.
- Support short commands such as `gi обновись`,
  `gi init D:\AI\general-instructions\`, `gi язык коммита: Russian`, and
  `gi язык коммита: English only`.
- Do not infer additional commit-message languages from the user's UI language
  or message language.
- If the user asks only to choose/select a commit language without naming
  languages, including `gi выбрать язык коммита`, ask which additional
  languages to enable or offer the supported list.
- If the user names languages explicitly, update
  `tools/project-memory/git-preferences.json` directly and summarize the new
  setting.
- When reporting this change, mention
  `tools/project-memory/git-preferences.json` as a plain path. Do not emit
  malformed markdown links or placeholder links.

## Verify

- Confirm an ambiguous request such as `gi выбрать язык коммита` asks for
  clarification instead of automatically enabling Russian.
- Confirm an explicit request such as
  `gi язык коммита: Russian` updates
  `tools/project-memory/git-preferences.json`.
