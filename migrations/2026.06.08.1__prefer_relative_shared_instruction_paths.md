# Migration: Prefer Relative Shared Instruction Paths

Version: `2026.06.08.1`

Apply this migration to copied instruction kits that should avoid stale
machine-specific shared-library paths such as missing Windows drive roots.

## Changes

- Prefer relative shared-instruction-library paths, `GENERAL_INSTRUCTIONS_HOME`,
  or the current shared library when resolving update and startup context.
- Do not treat a recorded absolute `update_check.shared_library_path` as
  authoritative when it is unavailable in the current environment.
- When creating or refreshing `tools/project-memory/instruction-kit.json`, avoid
  storing machine-specific absolute paths unless the user explicitly provided
  one. Prefer a relative path, an empty path with `GENERAL_INSTRUCTIONS_HOME`,
  or the explicit path from the current command.
- Update copied `tools/agent-start.ps1` logic so `GENERAL_INSTRUCTIONS_HOME` and
  the current shared library are checked before a saved metadata path.
- Update copied `tools/check-instruction-kit-updates.ps1` logic so running the
  command from a shared-instruction library can use `.` before falling back to
  environment or metadata paths.
- Replace fixed local examples such as user-profile helper paths and
  third-party workspace parents with environment-based or relative equivalents.

## Verification

- Run `git diff --check`.
- If updating `tools/project-memory/instruction-kit.json`, validate it with
  `ConvertFrom-Json` or another JSON parser.
- Confirm no generated logs, private paths, secrets, or unrelated project
  changes are included.
