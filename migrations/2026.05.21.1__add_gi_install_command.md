# Add GI Install Command

Migration ID: `2026.05.21.1__add_gi_install_command`

## Purpose

Add a short `gi` command for building the current project and producing an
installer artifact, with Inno Setup as the default packaging tool.

## Apply

- Treat `gi install`, `gi инсталл`, and `ги инсталл` as explicit requests to
  build the current project and produce an installer file.
- Treat obvious typo variants, such as `gi иснтлл`, as the same command when
  the user's installer intent is clear.
- If no installer program is named, use Inno Setup by default.
- If the user writes a program after the command, such as `gi install WiX` or
  `gi инсталл Inno Setup`, use that program as the preferred packaging tool.
- Before building, read project-local instructions, README, manifests, build
  scripts, packaging scripts, and installer configs such as `.iss` files.
- If the project has no clear build or installer contract, ask one concise
  clarification question instead of inventing installer configuration without a
  project-local basis.
- After a successful run, report the build result, installer artifact path, and
  checks performed.

## Verify

- Ask `gi install` in a project with an Inno Setup `.iss` file; confirm the
  agent builds the app and installer using the local project instructions.
- Ask `gi инсталл <program>` in a project with a different packaging tool;
  confirm the agent prefers the named program.
- Ask `gi иснтлл` in a clear installer context; confirm the command is treated
  as an installer build request.
- Ask `gi install` where no build or installer contract exists; confirm the
  agent asks a short clarification question.

## Rollback Or Conflict Notes

This migration adds a build/package command only. It must not grant permission
to read or modify files outside the current project root, publish releases, sign
artifacts, upload installers, or install software globally unless the user
explicitly asks and the required permissions are available.
