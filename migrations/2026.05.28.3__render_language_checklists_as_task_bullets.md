# Render language checklists as task bullets

Migration ID: `2026.05.28.3__render_language_checklists_as_task_bullets`

## Purpose

Make language selection prompts render reliably in chat clients that split
ordered-task checklist syntax across visual lines.

## Project Files

- Project-local `AGENTS.md`, runbooks, setup scripts, or task-manager adapter
  prompts that render language selection choices.

## Problem

Some chat renderers display ordered-task syntax such as `1. [x] English` with
the checkbox and label on separate visual lines. This makes the language
selection prompts look broken and harder to scan.

## Apply

- For unified project-language, commit-message language, and task-manager
  adapter prompts, keep the choices numbered but render each choice as a
  task-list bullet.
- Put the numeric choice inside the task-list label:

```markdown
- [x] 1. English
- [x] 2. Russian
- [ ] 3. Spanish
- [ ] 4. German
- [ ] 5. French
```

- Do not use ordered-task syntax such as `1. [x] English`.
- Keep the checkbox marker, number, and label on one physical Markdown line.
  Do not emit a standalone checkbox line followed by a separate numbered label
  line.
- Continue accepting numeric-only replies such as `1 2` against the visible
  numbers in the most recent checklist.

## Verification

- Ask `gi language` or `gi язык` without explicit languages.
- Confirm the checkboxes and language labels render on the same line.
- Confirm numeric replies still map to the shown choice numbers.
- Run `git diff --check`.
