# Add Context7 External Documentation Rule

Accepted version: `2026.06.16.11`

## Purpose

Add reusable guidance for Context7 and similar external documentation retrieval
tools. Agents need a safe way to use current public library, framework, SDK, and
API documentation without confusing it with project memory, service discovery,
task management, or local source truth.

## Apply To

- `AGENTS.md`
- copied project `AGENTS.md`
- `patterns/EXTERNAL_DOCUMENTATION_RETRIEVAL.md`
- copied project pattern files when present
- `INDEX.md`
- `CHANGELOG.md`
- `VERSION.md`
- `templates/instruction-kit.template.json`

## Required Changes

- Add Context7 as an optional external current-documentation retrieval layer
  when configured or explicitly requested.
- State that Context7 is for public library, framework, SDK, and API docs, not
  project memory, service discovery, task management, or current local source
  truth.
- Prefer project-local instructions and service guide/contract endpoints for
  project behavior.
- Prefer official OpenAI documentation workflows for OpenAI product questions.
- Forbid sending secrets, credentials, private source code, private business
  rules, user data, production data, telemetry, local paths, or project-memory
  contents to Context7 or similar external doc services unless explicit
  private-source configuration exists and the user approves the exact scope.
- Pin exact library IDs and versions when known.
- Verify current local source files before editing.
