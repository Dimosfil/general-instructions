# Commit Push Successful GI Updates

Migration ID: `2026.05.16.11__commit_push_successful_gi_updates`

## Purpose

Make successful `gi обновить` / `gi обновись` updates durable without requiring a
separate user command to commit and push the copied instruction-kit changes.

## Apply

- Treat successful `gi обновить` / `gi обновись` as an explicit request to commit
  and push the resulting instruction-kit update changes.
- Commit and push only when the current project is a git repository with a
  configured remote.
- Include only instruction-kit update files in the commit. Do not stage
  unrelated user or product changes.
- If unrelated/user changes are present, no git repository or remote exists, push
  fails, or conflicts remain, stop and explain the blocker.
- Keep the response scoped to the GI command.

## Verify

- Apply a pending GI migration in a clean git repository with a remote and
  confirm the agent commits and pushes the instruction-kit update.
- Repeat with unrelated dirty files and confirm the agent does not stage them
  and reports the blocker instead of forcing a commit.
